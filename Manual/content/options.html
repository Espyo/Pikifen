The following is an explanation of the player options in Pikifen.

All options saved by the game can be found in the /User_data/Options.txt file. You can edit this data file manually, though it's more comfortable to try to change these options in the options menu, or in the respective editor's options menu. Some options can't be found in either of those, since they are too niche or complex to show in-game, and need to be changed manually. The following options can be found in the file, though the player controls are also saved here, but aren't listed in this table for simplicity's sake:

OptionDescriptionDefaultPerformance impact?
ambiance_volume Volume for ambiance sounds, from 0 to 1. 1 No
anim_editor_bg_texture Background texture for the animation editor, if any. No
area_editor_advanced_mode If true, the area editor will use an advanced interface mode. false No
area_editor_backup_interval How often the area editor creates a backup of the current area, in seconds. 120 No
area_editor_grid_interval Space between each grid cell in the area editor. 32 No
area_editor_selection_transformation If true, a transformation widget will appear around the selection. false No
area_editor_show_circular_info If true, the radius and number of vertexes of a circular sector will be shown when dragging one. true No
area_editor_show_edge_length If true, the length of nearby edges will be shown, when dragging them or drawing them. true No
area_editor_show_path_link_length If true, the length of nearby path links will be shown, when dragging them or drawing them. true No
area_editor_show_territory If true, the territory radius of an object will appear around it, when selected. true No
area_editor_snap_mode Current snap mode used. 0 No
area_editor_snap_threshold The cursor only snaps when it is within these many pixels of something that can be snapped to. 80 No
area_editor_undo_limit How many steps can be saved in the undo history, at most. 0 disables the undo feature. 20 No
area_editor_view_mode View mode to use. 0 for standard, 1 for wireframe, 2 for height map, 3 for brightness map. 0 Yes
auto_throw_mode What mode auto-throwing is set to. 0 is off, 1 is via holding, 2 is via button toggle. 0 No
cursor_cam_weight When moving the cursor, how much does it influence the movement of the camera? 0 means none, 1 means just as much as the leader's position. You can use values beyond these ranges, but be warned that things will likely be weird. 0 No
cursor_speed The speed at which the cursor moves, used when controlling it without a mouse. This is in screen pixels per second. 500 No
draw_cursor_trail If false, the faint trail that follows the cursor will be removed, which could help with performance. true Yes
editor_mmb_pan In editors, should the middle mouse button pan (true) or reset the camera/zoom (false)? The right mouse button will do the other function. false No
editor_mouse_drag_threshold In editors, the mouse will only be considered dragged if it's moved at least these many pixels. 4 No
editor_open_nodes In editors, these nodes have been opened by the player. Next time they visit the editor, that node will already be opened. The nodes are named after the editor and the node itself, and are separated by semicolon. No
editor_primary_color In editors, if the player is using a custom style, this is the primary color to use for the widgets. 12 12 12 No
editor_secondary_color In editors, if the player is using a custom style, this is the secondary color to use for the widgets. 48 119 198 No
editor_show_tooltips In editors, should widget tooltips appear when the player places the mouse cursor over them? true No
editor_text_color In editors, if the player is using a custom style, this is the text color to use for the widgets. 255 255 255 No
editor_use_custom_style In editors, should a custom color style be used for the widgets? false No
fps Framerate cap. The higher it is, the smoother the gameplay is, but the heavier on performance the engine will be. 60 Yes
fullscreen Is the window fullscreen (true) or windowed (false)? Note that if the specified resolution doesn't work in fullscreen, the engine will revert the option to windowed mode. false Yes
joystick_min_deadzone Tilting the joystick this far or less will be considered the same as keeping the joystick in neutral. If your analog stick is wobbly, and you see actions happening in-game even thought you're not moving the analog stick, try increasing this value. 0.2 (20%) is a good value for most cases. 0.2 No
joystick_max_deadzone Tilting the joystick this far or more will be considered the same as holding the joystick fully in that direction. If you tilt your analog stick fully, but realize that the engine is responding like if you only tilted it a bit, try lowering this value. 0.9 (90%) is a good value for most cases. 0.9 No
leaving_confirmation_mode How the game should ask the player if they really confirm they want to leave the area. 0 is never, 1 is after 1 minute, 2 is always. 2 No
master_volume Master game volume, from 0 to 1. 0.8 No
max_particles Maximum number of particles to have on the world at once. If more particles than this need to be created, older particles are deleted. 1000 Yes
middle_zoom_level When pressing the camera button, the zoom level swaps between near, far, and this amount. 1.50 No
mipmaps true to enable mipmaps, false to disable them. Enabling them is recommended, and increases VRAM usage for slightly less GPU usage. Turn it off if you need RAM. true Yes
music_volume Volume for music, from 0 to 1. 1 No
resolution Width and height of the game window. The engine was made with a small-medium resolution in mind, although you can change the window size to anything you want. 1024 768 Yes
smooth_scaling If true, images that need to shrink or grow big will do so smoothly. Otherwise it will look pixelated, but it might help with performance. true Yes
true_fullscreen If true, fullscreen mode will really use a fullscreen window. If false, it will use a borderless window. Borderless window is recommended, since true fullscreen can cause issues when you Alt+Tab to a different window, as well as when a message box pops up (like browsing for a file in an editor). false No
ui_sfx_volume Volume for UI sound effects, from 0 to 1. 1 No
window_position_hack For some users, the game window does not appear, although the engine is definitely running fine. This is because it somehow gets placed out of the screen. Set this to true to force it to appear inside the screen. false No
world_sfx_volume Volume for in-world sound effects, from 0 to 1. 1 No
animation_editor_history_<number> List of recently-opened files on the animation editor. This does not need to be edited by the player, as the engine controls the list by itself. No
