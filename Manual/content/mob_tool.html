A tool object is anything that can be picked up by a Pikmin and have the Pikmin hold it until it is needed later. Most of the time, this is when the Pikmin is thrown.

Examples of tool types include the bomb rock, which can be carried around and explodes when dropped, and the mine from Bingo Battle, which sticks to surfaces and objects and explodes when a leader draws near.

To create a new type of tool, please follow the making object types tutorial.

Overview

Tool object types dictate what happens when the Pikmin picks them up, and what happens when they are dropped. They also specify when they can and cannot be picked up, be it by Pikmin or by enemies, and when exactly it is that they get activated. Their behavior is specified by the user-made script.

Tool type objects are hardcoded in the engine to be chased out by the Pikmin (when appropriate), picked up, released normally, released gently when the Pikmin needs to enter an Onion, and optionally, get stuck to an object.

Animations

Tool object types do not need any specific animations, only the ones that you call in the script.

Script

All of the tool type object's behavior is defined by the user-made script. It is recommended that you use the set_holdable action at some point to make it so Pikmin can hold the tool, and listen out to the on_held and on_released events, to know when the tool has been picked up, and is ready to be used. The engine changes the gentle_release script variable to true when the Pikmin releases the tool "gently", i.e. when the Pikmin needs to go inside an Onion, and sets it to false otherwise, so you can use this to distinguish whether to leave the tool idling on the floor, or to activate it.

Some script variables control special behavior, like the tool's starting team. To know which ones exist, you can go to the area editor, select a tool, and read the tooltip of each variable.
