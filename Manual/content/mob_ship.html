Object category – ship

A ship type refers to any vessel that the Pikmin can deliver treasure, berries, etc. to. They may also house Pikmin inside, and leaders may also heal on them.

Examples of ship types include the S.S. Dolphin, Hocotate ship, the Hocotate ship's pod, etc.

To create a new type of ship, please follow the making object types tutorial.

Overview

Every ship has a beam next to it; this is where the Pikmin and leader interact with the ship proper. They can also store Pikmin inside, like Onions, though they are incapable of generating Pikmin seeds.

Ship type objects are hardcoded in the engine to sit there and accept delivered items, to heal leaders if possible, and to allow the player to manage the Pikmin inside if possibe.

Properties

Besides the global object type properties, a ship type has the following properties.

Mandatory

None.

Recommended

  • can_heal: If true, that means that leaders can heal their spacesuits using this ship, like the S.S. Dolphin. Defaults to false.

Optional

  • leg_body_parts: Names of the body parts that make up the ship's "legs", separated by semicolon. This makes more sense if you follow the logic of the way an Onion uses this property.
  • pikmin_types: Names of the types of Pikmin this ship can manage, separated by semicolon. Leave empty if the ship cannot manage Pikmin.
  • pikmin_enter_speed: When a Pikmin is ordered to enter the ship, it will climb up a "leg" (see above). This indicates the speed, in ratio per second. See the track object category for more information on what this means. Defaults to 0.7.
  • pikmin_exit_speed: Same as pikmin_enter_speed, but for popping out of a ship "leg". Defaults to 2.

Animations

A ship type needs to have the following animations:

  • idling: Idling; used for everything.

Script

Ship types have hardcoded scripts, so the only thing you can do is enhance them.
