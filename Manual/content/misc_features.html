This page will describe several miscellaneous features in Pikifen that don't quite fit in other pages.

Object links

You can make one object in the area have a link towards another object. This allows one object to send messages to its links, with the appropriate script actions. It also makes it possible for the engine to know how certain special objects should function, when they need to know data about a second object. Connections are one-way, so if you want object A to send messages to object B and vice-versa, you will need to create a link from B to A as well.

To link an object to another, you can either link them in the area editor, link them via the link_with_focused_mob object script action, or do the linking when an object is spawning another. In the latter case, you can specify if you want the link to be from the spawner object to the spawned object, the opposite, or both.

Markup

When writing some text to show in-game, some features of the engine allow you to write special tags, which will be replaced with something else in-game, like a line break or the icon of a player input. These tags always start with a backslash (\), so if you want the engine to show an actual backslash in-game as opposed to trying to understand some tag, write \\ instead. Unless otherwise specificed, features that support markup will also automatically break into the next line when the current line is too long. The following features support markup:

  • An area's description, to be shown on the area selection menu.
  • Mid-game message boxes, like the ones shown by the show_message_from_var object script action (commonly used by the likes of signs).
  • The pause menu's help tidbits.

The following tags exist:

  • \\: Inserts an actual backslash (\).
  • \n: Inserts a line break, meaning the text will end the current line and go to the next.
  • \k: Inserts an input's icon for a given action. This will automatically show the key, button, etc. that the player has mapped to the action you specify, and can even show "(NONE)" if none is bound. The name of the action needs to be placed between two of these tags, and you can use spaces between them to make it easier to read. So for instance, typing \k throw \k will show the input the player has currently assigned for throwing, so likely the icon for a left mouse button. The names of each action are pretty obvious, like throw, move_up, or lie_down, but you can open the options file and check the properties that save the player's controls for a full list.

System info

By pressing F1, you can toggle the system info visualizer. This is a good way of telling how smoothly or slowly Pikifen is running, and as such, it's a good way of helping you gauge if the content you're making is computationally heavy or not.

The 9 main numbers at the left are all framerate-related.

  • The "Now" column is what the measurements are on this exact frame.
  • The "Average" column is the average of the measurements in the last half-second.
  • The "Target" column is what Pikifen is targetting, which matches the framerate set in the options (typically 60 FPS).
  • The "FPS" row is the framerate (the amount of frames per second), meaning the number of frames the game processes in a second. The numbers in this row represent what's really happening in terms of what gets processed and drawn to the screen on a timer, so they cannot go higher than the framerate limit (again, typically 60 FPS).
  • The "FPS uncap." row is what the framerate would be if it weren't limited.
  • The "Frame time" row is how long the frames are taking to be processed, in seconds.

To the right is a chart representing the last few frames of gameplay. Each column in the chart represents one frame, and is painted from the top down. The farther down it reaches, the higher the FPS was on that frame. If it reaches the vertical bar at the bottom, that means that frame was processed in the appropriate amount of time set in the framerate settings, but if not, that means that frame was a bit slower.

The rest of this visualizer gives you some miscellaneous stats about what's going on in the area, as well as general information about Pikifen.
