The custom object category is for any object that doesn't require a specific behavior. This is where you'd place a completely custom object, like say, a rubber ball that bounces around when a Pikmin is thrown at it, but is not meant to do anything specific, like be carried. This category also contains popular, but simple objects, like eggs, gates, obstacles, Pellet Posies, boulders, etc.

To create a new type of custom object, please follow the making object types tutorial.

Overview

The behavior of custom type objects is controlled entirely by the script.

Properties

Besides the global object type properties, custom object types do not have any specific properties.

Animations

Custom object types do not need any specific animations, only the ones that you call in the script.

Script

All of the custom type object's behavior is defined by the user-made script. There are no recommended states, events, or actions.

Some script variables control special behavior, like the object's starting team. To know which ones exist, you can go to the area editor, select a custom-type object, and read the tooltip of each variable.
