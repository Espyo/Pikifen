The following is a general changelog for Pikifen. This changelog will only list the most notable differences between each version of the engine; for a more detailed list of changes throughout the development period, check the commit log on GitHub.

Some versions may be unable to read content created in older versions. If you want to keep your content from one version to the next, read the "Content making changes" list in that update's section.

Versions are numbered after the format major.minor.revision. The major number increases when the project is radically different from the major version before it. The minor number increases every time there are a bunch of changes. The revision number increases when there are bug fixes on the revision before it, and does not break compatibility unless that's necessary to fix something.

0.25.0

Under development

Gameplay changes

  • Added levers. Interact with a leader or throw a Pikmin to power or stop powering devices.
  • Added buttons. Interact with a leader or throw a Pikmin to power devices for a split-second.
  • Changed how some objects push against each other. Now, leaders and Pikmin can squeeze through them to get unstuck. This does not apply to more "solid" objects like gates.
  • Changed the range for an idle Pikmin to be considered "bumped" by the active leader, so they can be called from farther away.
  • Changed the way the Yellow Wollywog works to make the fight easier and a bit more dynamic.

Content making changes

  • Added the pushes_softly property to mob types.
  • Added support for the thus far-unused maturity_power_mult property of the game configuration.
  • Added the idle_bump_range property to the game configuration.
  • Added the ability for a sector to reuse the same vertex. Before, it just considered the sector broken.
  • Added the ability to create sub-sectors. This happens if one sector's polygon ends up getting split into two. While not recommended, it stops the maker from receiving warnings and from the sector to be considered broken.
  • Added the ability to keep drawing a new sector, for when you try to split a sector but the engine considers that split useless.
  • Added a redo feature to the area editor.
  • Added three new controls: custom_a, custom_b and custom_c. These do nothing on their own, but can be detected through leader scripts. (Thanks WanderingLetterS)
  • Added the on_input_received object script event.
  • Added the input_name and input_value properties to the get_info object script action.

0.24.0

July 14th, 2023 – the "new areas" update

Gameplay changes

  • New areas:
    • Added Tutorial Meadow. If you never played an area before, you will be prompted to start here.
    • Added Cryptic Atrium. A medium-short puzzle stage with pressure plates and automatic gates.
    • Added Playful Yard. A reincarnation of the old "Play" area, with a more playful motif, better design, and as a proper mission.
    • Added Crash Landing. A difficult quick mission to get all treasures before time runs out. (Thanks Helodity)
    • Added DEV Lab. It showcases pretty much everything Pikifen has to offer.
    • Removed Play in favor of Playful Yard.
  • New objects:
    • Added orb signs, which work like signs but are generic orbs instead of wooden planks.
    • Added geysers, a type of bouncer that only leaders can use.
    • Added carriable rocks, which can be carried around and dropped to weigh on something.
    • Added pressure plates, which trigger some other object to do something whenever they have enough weight on them.
    • Added automatic gates, which lift in the air whenever they are powered by things like a pressure plate.
    • Added carry targets, which allows other things to be carried to it for no effect.
    • Added some treasures: a fidget spinner, a yo-yo, and a jigsaw puzzle piece.
  • Misc:
    • Added images to the help page of the pause menu. (Thanks erdinvark)
    • Added a random area button to the area selection menu.
    • Added mission statistics for the total of clears, total of platinum medals, and total combined sum of score points.
    • Changed the way paper bags work to make it so leaders can't be thrown on top, and so that Pikmin can't accidentally be thrown over them to the other side.

Content making changes

  • Editor improvements:
    • Added a bunch of menu bar items across all editors.
    • Added a toolbar button to the area editor to toggle preview mode. Advanced interface mode only.
    • Added the ability to lock to an axis while dragging, if Ctrl is held.
    • Added the ability to add to the selection when dragging a selection box, if Ctrl is held.
    • Added the ability to copy a sector, edge, object, or path link's properties, so they can then be pasted elsewhere.
    • Added some shortcut operations to the sector height widget as a right-click context menu.
    • Added settings for the area editor's path preview tool. Plus some more detailed information about the path calculation.
    • Changed the unsaved changes warning to be a dialog like in most programs. The idea of the old pop-up was to require fewer clicks and mouse movement when you were sure you wanted to not save, but some actions are hidden behind menus which forces you to go there twice. The dialog approach comes with many quality-of-life tweaks that make it an overall better feature.
    • Changed the way copying textures from a different sector works. Instead of selecting the source, then hovering the mouse over the destination and pressing T, you now copy the source sector's properties, then paste just the texture on the selected sectors.
    • Changed what happens when the user is drawing a new sector and clicks on an edge or vertex. Instead of throwing an error, the editor now changes the drawing to a sector split drawing, and lets the user continue drawing from the opposite end.
    • Changed the area editor's saving procedure to no longer need to boot the user back to the main panel.
  • Others:
    • Added the hud maker tool.
    • Added the path_info maker tool.
    • Added the custom_carry_spots property to mob types.
    • Added the power_goal script variable to group tasks.
    • Added the goal_number script variable to scales.
    • Added the stay value to a resource type's delivery results.
    • Added the linked_mob_matching_type value to a resource type's carrying destinations.
    • Added the area_image_padding property to the maker tools.
    • Added the ability for dead enemies to drop objects via a "stored inside" feature in the area editor. This allows the engine to detect things like treasures inside enemies for the purposes of missions.

Breaking changes

  • Removed the ability for dead enemies to drop objects via the spoils and random_pellet_spoils script variables, in favor of the "stored inside" area editor feature. Areas with such objects need to be adjusted.
  • Changed the following obstacles to have a consistent 192px size: electrical wire (also now doesn't push), paper bag (also now blocks paths), cardboard box, wood bridge.
  • Changed the spritesheet for player input icons (formerly control icons). All properties and files changed from control_icon to player_input_icon.
  • Changed the show_hud_controls option to show_hud_input_icons.
  • Changed the gameplay GUI item names that had button in them to instead have input.
  • Changed the default value of an object type's can_walk_on_others properties to false.
  • Changed the name of some texture files: Bark → Wood_bark. Bridge → Wood_bridge. Bridge_rail → Wood_bridge_rail. Metal → Metal_rusted. Stump → Wood_stump.

0.23.0

December 25th, 2022 – the "Mission Mode" update

Gameplay changes

  • Missions:
    • Added mission-type areas. Gameplay during missions usually involves reaching a goal, avoiding some fail conditions, and getting medals based on several scoring criteria.
    • Added info about the area that is currently selected in the list, in the area selection screen.
    • Added a mission information tab to the pause menu, if you're playing on a mission.
    • Added a "play" button and a "make" button for the main menu, which show the list of area types that can be played, and the list of editors, respectively.
    • Changed the results screen to better suit a mission's completion screen.
  • Lifetime statistics:
    • Added lifetime statistics, accessible from a menu in the main menu. These are saved onto the disk.
  • Others:
    • Added the ability to play when there are no available leaders, or even if there were none to begin with. There isn't much to do in this state other than move the camera, however.
    • Added the ability for the camera to be influenced by the group and the cursor. Added an option to control how much the cursor influences it.
    • Added a confirmation message for the pause menu, shown when you want to leave. This can be changed in the options. (Thanks Neo)
    • Changed the pause menu with various improvements.

Content making changes

  • Area editor improvements:
    • Added a history of opened folders.
    • Added a "gameplay" panel, with the starting spray amounts (with a better interface), as well as the new mission stuff.
    • Added description, tags, thumbnail, and difficulty properties for areas.
    • Added a show path link length option. (Thanks Nataly)
    • Added a show circular sector info option. (Thanks Neo)
    • Added a basic/advanced interface mode option.
    • Added the ability to choose what type and label new path links will have as you're drawing them. (Thanks Nataly)
    • Added highlights to a sector, mob, etc. when you hover your mouse over it. (Thanks Helodity)
    • Added toolbar buttons to swap to the layout, objects, paths, or details mode. Advanced interface mode only. (Thanks Nataly)
    • Added the H shortcut key to change a sector's height.
    • Added the T shortcut key to copy a sector's texture.
    • Added the L, O, P, D shortcut keys to swap to the layout, objects, paths, and details mode. Advanced interface mode only. (Thanks Nataly)
    • Added liquids to the area preview feature.
    • Added the ability for areas to specify at what point of the in-game day they start, as well as day time speed or (for missions with time limit) the time of day they end.
  • Maker tools:
    • Added the Ctrl+F12 shortcut to change the auto-start maker tool's data to match the current mode and currently open content.
    • Added a "show collision" maker tool. (Thanks Helodity)
  • New properties:
    • Added a points property for enemy types.
    • Added a states_ignoring_hazard property for scripts.
    • Added a hold_offset_vertical_distance property for a mob type's children data.
    • Added a custom_category_name property for mob types. With this, the pre-packaged mob types are now better organized in the editors.
    • Added a can_walk_on_others property for mob types.
    • Added a knocked_down_duration and a knocked_down_whistle_bonus property for Pikmin types.
    • Added an auto_shrink_smallest_radius property for pile types.
    • Added an interval_deviation property for particle generators.
  • GUI editor:
    • Added a GUI editor. This should make it easier to position and resize GUI elements in menus and HUDs.
  • Others:
    • Added the ability to lock the center (as opposed to the opposite side/corner) by holding Alt, when you are dragging a side/corner handle of a transformation widget in an editor's canvas.
    • Added script variables to decoration objects to allow them to disable the random property changes.
    • Added the gentle_release script variable to tool objects so they can know if the Pikmin released it gently (i.e. the Pikmin is entering an Onion) or not.
    • Added logic for rectangular objects to be able to push other rectangular objects. (Thanks Helodity)
    • Changed the duplicate mobs area editor keyboard shortcut from D to Ctrl+D, since D is now used to go to the details mode.
    • Changed keyboard shortcuts made for the editor canvases to make them now work anywhere so long as you aren't writing in a text box. (Thanks Neo)
    • Changed drag buttons in editors to only need a single click in order to enter text input mode.

Custom content compatibility warnings

  • Revamped the following GUIs:
    • Changed the main menu's GUI.
    • Changed the area menu's GUI.
    • Changed the results screen's GUI.
    • Changed the pause menu's GUI.
  • Others:
    • Changed the location of area folders, where simple areas used to be in Game_data/Areas/<folder>, but must now be moved to Game_data/Areas/Simple/<folder>. If you care about auto-backups and reference image data, do the same to the area-related stuff in the User_data folder.
    • Removed the day_minutes_per_irl_sec field from the config file since now each area controls its speed.
    • Removed the system asset for the Pikmin silhouette, in favor of two leader silhouettes.
    • Added two leader silhouette graphics to replace the Pikmin silhouette image.
    • Added a standard_leader_height and a standard_leader_radius game config property.

0.22.0

July 5th, 2022 – the "polish" update

Gameplay changes

  • Added a help page to the pause menu, with gameplay information, tips, controls, Pikmin type explanations, and more.
  • White Pikmin poison now applies a status effect that drains health over time. Enemies still end up losing the same amount of health in the end.
  • Pikmin are now better at detecting gates and other rectangular objects.
  • Pikmin can now circle around enemies when in combat.
  • Bridges now grow in chunks, and the point Pikmin deliver fragments to, or the position of the pile of sticks, now moves along with the bridge.
  • Misc. graphical enhancements:
    • All gates have received graphical improvements.
    • Decorative objects no longer cast light in blackout weather.
    • When Pikmin deliver an object to an Onion, ship, etc., the sucking animation is now more dynamic. Also, added a tossing animation for delivering bridge fragments to bridges.
    • Improved the look of the ship's beam and the leader's whistle. The pretty_whistle option has been removed. (Thanks Helodity)
    • HUD elements now animate when things change, and slide off-frame when menus are entered in-game. (Thanks Helodity)
    • In-game message boxes are now much more animated.
    • Health wheels and fraction numbers now appear and disappear smoothly, and are more animated in general.
    • Enemies now squash and stretch when damaged.
    • Objects being carried now sway.
    • Icons for controls have been improved.
  • Fixed leaders being able to grab group members from the other side of walls or objects with collision, like gates.

Content making changes

  • Bridges are now made up of objects. This should make the process of placing bridges in an area much simpler, since there's no need to fiddle around with sectors. Areas with bridges need to be updated to use this new bridge format.
  • Status effects now have a health_change property, which controls how much health the object loses per second in an absolute value, instead of it being a ratio of total health.
  • Spike damage types now have a status_to_apply property, which allows the damaged object to receive a status effect.
  • Object types now have a status_vulnerabilities block, which works like the other vulnerability blocks, but controls damage taken from status effects.
  • Object types now have a terrain_radius property, which controls their collision against walls.
  • Object types now have a description property, which can be used to briefly describe it. This appears in the area editor, and Pikmin types make use of this for the help page of the in-game pause menu.
  • The tooltip of an object type's area editor property now word-wraps automatically. If you relied on line breaks (\n) to do this before, you may want to adjust it.
  • The set_animation script action now accepts the options random_time and random_time_on_spawn, to let the animation start on a random time, or start on a random time but only if the mob just spawned.
  • Added the on_receive_delivery object script event.
  • Added the health_ratio option to the get_info object script action.
  • Added the health script variable.
  • Added the max_health script variable.
  • Added the bright_circle and bright_ring system assets.
  • The area editor now also shows the terrain radius of selected objects (if the "Show territory/terrain radius" option is on).
  • Converters now use an animation called bumped instead of bumping, for consistency's sake.
  • System assets no longer have icons for mouse-related controls, but instead have one spreadsheet for many special control icons.
  • The gameplay HUD received all sorts of new items, and some have been renamed to be more consistent.
  • Areas and animations now save the engine version they were edited in. This will likely be used in the future to allow the engine to automatically port older content to newer versions.
