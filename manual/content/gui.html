All menus and HUDs (heads-up displays) in Pikifen can be customized. Most of them only allow you to change the position and size of the different items, but some may allow for more in-depth customizations. Each menu or HUD has a GUI (graphical user interface), and each GUI definition is a piece of content, so you should know how content works in the engine in general before starting. Each one has its own data file, located in game_data/<pack>/gui.

To change the position and size of elements, you can just use the GUI editor, accessible from the main menu. To change other things, you will need to edit the data files by hand.

This page explains how the editor works, as well as list all the different menus and HUDs that can be customized.

GUI editor

The GUI editor is a simple tool that allows you to change the position and size of each item in a menu/HUD.

Opening a GUI definition file

When you open the menu, you'll be asked what GUI to edit.
You can access the GUI editor from the main menu's "make" page. Once you do, you'll be shown a list of menus and HUDs whose GUIs you can edit.
The editor only knows each GUI definition's internal name, but these are usually obvious.
Because the editor is made to be simple and flexible, allowing the customization of any GUI that the engine has or will have, it doesn't actually know anything about specific menus, it just knows the names of the data files inside the GUI definitions folder. So this dialog will show you the text file names, i.e. the internal names, instead of the menu or HUD names. Still, these names are pretty obvious, and if you're ever in doubt, check the GUI definitions section of this page, since that lists what each menu's file is.

Canvas and coordinates

The left side of the editor represents the player's game window.
Once you open a file, you'll see a green square in a black void in the canvas, on the left half of the editor. This square represents the player's game window, and is where all the items are positioned.
GUI item positions and sizes aren't specified in pixels.
You may find it weird how the green shape in the canvas is a square, since almost nobody uses a square game window. The trick with this editor, and with the way GUI item positions and sizes work, is that you don't specify exact coordinates for where they should go, or how big they should be. If you want to horizontally center a certain item on-screen, you can't just say its center's X coordinate is 512 pixels or something. While 512 pixels may look centered in your game's window, it may look completely off in the window of somebody with a different size – remember each player can set the game's window size in the options menu!
Positions and sizes are measured in screen percentages.
Instead of pixels, you have to specify percentages of the screen. To center something horizontally, you specify its center's X coordinate as 50%. To make it flush to the top of the screen, its Y coordinate should be 0%. To make it take up a third of the screen horizontally, its width should be 33%. And so on. So, in reality, the green square you see in the canvas is just a representation of percentages, and while it's a square here, the real game window will likely be stretched differently depending on each player's resolution. Still, the idea is for items to be placed in the portion of the screen they work best at, for items to look good together, for items to look nice when spaced evenly apart, etc., so exact pixels don't really matter.
The mouse cursor's position can be seen in the status bar.
As you move the mouse cursor about, you can see where it is positioned in the green square by checking the right side of the status bar, located at the bottom of the editor.

Panel

The panel on the right lists all items.
The right side of the editor has a panel that includes a list of all the items in the current GUI definition, as well as their internal names.
In the list, checkboxes specify whether an item is visible.
You can click on an item's checkbox in the list to make it visible or invisible. If it is invisible, it won't show up in the canvas nor in-game.
Click an item in the list to select it.
When you click an item (not its checkbox), you will see some information about that item appear below the list in the panel. You will also see some handles on the canvas, surrounding that item. Those handles are explained later.
Edit the selected item's position or size in the widgets below the list.
The numbers in the "center" and "size" widgets represent screen percentages, as explained above. If instead you would like to specify where an item's top-left corner is, or bottom-right corner, there are some widgets to adjust things that way.

Interacting with the canvas

Click an item's bubble in the canvas to select it.
Instead of selecting an item through the panel's list, you can click on it in the canvas to select it. Doing so will also correctly select the corresponding item in the list.
When items overlap, keep clicking to select different ones.
If you click somewhere where two items overlap, the editor will select one of them. But if you click there again, it will select the other one. You can keep clicking to change which item is selected, making it a good way to find the exact item you need when you've got a lot of them in a tight space.
Drag the central handle of the selected item to move it.
Once an item is selected, handles appear allowing you to shape it in the canvas. This makes things much easier than tweaking numbers manually in the panel. The central handle moves the item around.
Drag the other handles to resize it.
The handles at the edges and corners of the selected item allow you to resize it, be it horizontally, vertically, or both. You can hold Alt to lock the center, instead of locking the opposite edge or corner.
Hold Shift to disable snapping.
When you move the handles, they snap to the nearest grid point. This helps you keep things aligned. If you don't want that, you can hold down the Shift key to disable snapping. You can also disable snapping by using the button in the toolbar that controls the snapping mode.
Hold Ctrl to snap to horizontal or vertical axis.
By holding Ctrl, you can move something just up or down, or just left or right.

Putting it all together

When you're done, save with the button in the toolbar.
When you press the save button, the GUI definition gets saved to a file on the disk, and if you want, you can now quit the editor (with the button in the toolbar), go to where the relevant menu or HUD is, and check it out in action. Are things not quite what you expected? Here are some things to keep in mind:
The editor can only uses an item's internal name and draw a vague shape representing it.
Because each GUI item can behave in its own way, and look however it pleases, it is impossible for the GUI editor to show them in the canvas. Something like an icon in a menu might be simple, but something like the Sun Meter is composed of many individual moving parts. So you're going to have to use your imagination when using the editor. Likewise, the editor doesn't know what each GUI is doing, it just knows what items exist (and what their positions and sizes are) because that information is in the data files. So you'll have to get used to handling the internal names of items instead of something more descriptive. Though again, if you're ever unsure what an item is, check the GUI definitions section of this document.
An item's size only specifies its maximum size.
If you give an item a large width, but a small height, you may be surprised to find that, in-game, it doesn't look stretched out, and in fact looks pretty tiny. There is no one rule that controls what each item looks like – every single item in a menu or HUD behaves however it wants. The size properties in the GUI definition simply dictate the maximum width or height that the item is allowed to have. Normally, a GUI item will never be larger than the width or height you specified, though it can be smaller.
GUI items can animate, including moving around.
When gameplay starts, the HUD moves into view from the edges of the screen. When you pause, the menu slides into view from off-screen. A GUI's definition does not include any information about this, since that's all controlled by the engine. So don't be surprised that a GUI item can be in a position different from the one you specified – it's just moving into place! Likewise, some counters animate by growing their text whenever their number changes. This can result in them being bigger than the specified size, even if it's only temporary. If you're absolutely worried that this might make it bump into another item, remember to give the item some extra breathing room in the editor.
If you want an item gone, mark it as invisible instead of placing it off-screen.
While a GUI item can be partially off-screen if you want, they shouldn't be entirely off-screen. Because GUI items can animate and move during transitions, an item that is placed off-screen might come into view during these times, which can confuse the player. Plus, having these items off-screen will still make the engine waste resources on processing them, even if they're practically useless in-game. Instead, untick its checkbox in the list of items, in the panel.

GUI definitions

Each of these sections refers to a menu/HUD, and explains what GUI definition controls it, the internal names of each item, and what properties can be changed in the GUI definition's data file. The positions block in a data file controls the position and size of each item in that menu/HUD, but as explained before, you're better off using the editor for these.

Main menu

The main menu, entered when you start the game.

  • Internal name: main_menu
  • Items:
    • play: Play button.
    • make: Make button.
    • options: Options button.
    • stats: Statistics button.
    • discord: Discord server invite button.
    • github: GitHub page button.
    • exit: Exit button.
    • tooltip: Tooltip of the selected menu item.
  • Logo: The logo that is made of Pikmin can be customized in the logo block.
    • min_screen_limit: Top-left coordinates of the logo's placement on-screen, in screen width or height percentage.
    • max_screen_limit: Same as above, but for the bottom-right.
    • pikmin_max_speed: Each Pikmin's speed is random. This is the maximum speed possible, in pixels per second.
    • pikmin_min_speed: Same as above, but minimum possible.
    • pikmin_speed_smoothness: The closer to their destination, the more the Pikmin slow down, based on this multiplier. 0.08 is a good number.
    • pikmin_sway_amount: How many pixels a Pikmin sways left or right when standing still.
    • pikmin_sway_max_speed: Each Pikmin's swaying speed is random. This is the maximum speed, in radians per second (the Y part is ignored when swaying).
    • pikmin_sway_min_speed: Same as above, but the minimum speed.
    • pikmin_size: Pikmin size, in screen dimension percentage. For instance, 3 2 means each Pikmin cannot be any larger than 3% of the screen width nor 2% of the screen height.
    • pikmin_types: This block specifies the types of Pikmin you'll encounter in the title logo. Except every type is simply defined by the image of its flower. In this block, every line is a type, in the format character = image, where character is a text character representing this Pikmin type in the logo map (see below), and image is the internal name of the image in the game_data/<pack>/graphics folders.
    • map: A representation of the logo. The Pikmin are organized in a grid, and that grid is spread to fit in the limits specified above. Every line in this block represents a line in the grid, with every character matching a cell in that line. A character represents a Pikmin type or a blank space, if it's a character specified in the pikmin_types block or a dot (.), respectively.
  • Icons to the left: Use this to indicate whether the icon on each button should be on the left side (true) or the right side (false).
    • play: Play button.
    • make: Make button.
    • options: Options button.
    • statistics: Statistics button.
    • quit: Quit button.

    • idle_pikmin_number: Idle Pikmin counter.
    • field_pikmin_label: Label for the field Pikmin counter.
    • field_pikmin_number: Field Pikmin counter.
    • cursor_info: Information about whatever the radar cursor is on.
    • instructions: Some basic instructions on how to use the radar.
    • tooltip: Tooltip of the selected menu item.
  • Files: You can specify the internal name of the bitmap files some of the radar icons should use in the files block.
    • pikmin: Icon representing a Pikmin.
    • treasure: Icon representing a treasure.
    • enemy_alive: Icon representing a living enemy.
    • enemy_dead: Icon representing a dead enemy.
    • leader_bubble: Bubble to place around a leader's icon.
    • leader_x: X imposed on top of KO'd leader icons.
    • obstacle: Icon representing a generic obstacle that blocks paths.
    • onion_skeleton: Skeleton part of the icon that represents an Onion. This is the part of the icon that doesn't change color, i.e. the legs and other details.
    • onion_bulb: Bulb part of the icon that represents an Onion. This is the part that changes color depending on the Pikmin it can manage.
    • ship: Icon representing a ship.

Pause menu – status page

The Pikmin status page of the pause menu.

  • Internal name: pause_status
  • Items:
    • header: Header text saying "STATUS".
    • left_page: Button to go to whatever page is to the left.
    • left_page_input: Icon of the player input that changes to the left page.
    • right_page: Button to go to whatever page is to the right.
    • right_page_input: Icon of the player input that changes to the right page.
    • line: Horizontal line separating the header and the menu.
    • continue: Continue button.
    • list_header: Row for the header of the table, which details what each column is.
    • list: The table proper.
    • list_scroll: Scrollbar for the list.
    • totals: Row containing the grand totals.
    • instructions: Some basic instructions on how to use the radar.
    • tooltip: Tooltip of the selected menu item.

Results menu

The menu shown when players end an exploration/mission and get some results.

  • Internal name: results_menu
  • Items:
    • area_name: Name of the area.
    • area_subtitle: Area subtitle or mission goal.
    • goal_stamp: A stamp indicating whether or not the player reached the mission goal.
    • end_reason: Reason as to why the mission ended, if any.
    • medal: Mission medal that the player got, if any.
    • medal_reason: Reason for the medal. If applicable, includes the final score.
    • stats_label: Label for the stats box.
    • stats: Box with several statistics.
    • stats_scroll: Scrollbar for the stats box.
    • retry: Retry button.
    • continue: Keep playing button.
    • pick_area: Pick an area button.
    • tooltip: Tooltip of the selected menu item.
