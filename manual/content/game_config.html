Some settings about how the game works can be configured in a data file. It can be found in game_data/<pack>/misc/config.txt. Create it if it does not exist. This file a piece of content, so you should know how content works in the engine in general before starting.

The following properties exist, split into blocks:

Aesthetics, general

Block name: aesthetic_general.

carrying_color_move Color to use for the carrying numbers when an object that is meant for the ship is being carried. Color 255 255 255
carrying_color_stop Color to use for the carrying numbers when an object is not moving. Color 96 192 192
gameplay_msg_char_interval Time interval between characters when displaying a gameplay message, in seconds. 0 shows the entire text instantly. Number 0.03
mouse_cursor_spin_speed How quickly the mouse cursor's icon spins, in degrees per second. Number 180
no_pikmin_color Color that represents the absence of Pikmin. Color 168 190 230

Aesthetics, radar

Block name: aesthetic_radar.

background_color Color to use for the background in the radar. Color 32 24 0
edge_color Color to use for sector edges in the radar. Color 32 24 0
highest_color Color to use for the highest sector of the area in the radar. Color 200 200 180
lowest_color Color to use for the lowest sector of the area in the radar. Color 80 64 0

Carrying

Block name: carrying.

speed_base_mult When an object is being carried, its movement speed depends on the carriers' average speed (and more). With 0 carriers, the average speed is multiplied by base_mult, and with all spots taken, it's multiplied by max_mult. This property controls the base value. In other words, this controls how high the slowest speed of a carried object is. Number 0.3
speed_max_mult Same as above, but the max_mult. In other words, this controls how high the fastest speed of a carried object is. Number 0.8
speed_weight_mult For every 1 unit of weight, decrease the carrying speed by this much. Number 0.0004

General

Block name: general.

game_name Name of the fan game. Text
game_version Version of the fan game. Can be anything, but the preferred format is something like "1.2.3". Text

bad Red color for "bad" things. Color 226 112 112
focused_item Color of the currently focused GUI item. Color 87 200 208
gold Gold color for grandiose things. Color 255 242 0
good Green color for "good" things. Color 96 226 80
page_change Buttons that change pages. Color 188 230 230

small_header Small, not-very-important headers. Color 188 230 230

Leaders

Block name: leaders.

group_member_grab_range A leader can grab the closest group member only if it is within this distance. Number 128
next_pluck_range When auto-plucking, a leader will only choose the next Pikmin if it is within this distance. Also affects the range at which idle leaders will go pluck. Number 200
onion_open_range Leaders can open an Onion if they're within this range from the center of the Onion object. Number 24
order Like pikmin > order, but for the leaders. Text list
pluck_range A leader can pluck a Pikmin only if it is within this distance. Number 32
standard_height Like standard_pikmin_height, but for the leaders. Number 46
standard_radius Like standard_pikmin_radius, but for the leaders. Number 16

Misc.

Block name: misc.

day_minutes_end Same as above, but for when gameplay ends. Number 1140
day_minutes_start Time of day when gameplay starts. This follows a format similar to real life, so 0 is midnight, 60 is 1 AM, 120 is 2 AM, 420 is 7 AM. This is only used for the Sun Meter HUD item. Number 420
spray_order Like pikmin > order, but for sprays. Text list

Pikmin

Block name: pikmin.

chase_range Pikmin will chase after enemies unless they're this far away. Number 200
idle_bump_delay In order for an idle Pikmin to be bumped, it must be away from the active leader for these many seconds first. Number 5
idle_task_range Minimum distance between a Pikmin and a task before the Pikmin can take the task, when the Pikmin is idling. Number 50
maturity_power_mult Every maturity level, add this much to a Pikmin's attack power, as a ratio (e.g. 0.1 adds 10% more). Number 0.1
maturity_speed_mult Same as maturity_power_mult, but for the movement speed. Number 0.1
order Preferred order to sort Pikmin with. Used when swapping Pikmin, dismissing them, organizing them in an Onion, etc. Types you do not specify will be sorted alphabetically and go to the end of the list. Text list
standard_height The height of a "standard" Pikmin, i.e. a Pikmin that's not shorter or taller than most. Used to help calculate some physics logic. Number 24
standard_radius The radius of a "standard" Pikmin, i.e. a Pikmin that's not skinnier or fatter than most. Used to help calculate some physics logic. Number 5
swarm_task_range Same as above, but for a Pikmin in the group during swarming. Number 3

Rules

Block name: rules.

can_throw_leaders Whether it's possible for leaders to throw other leaders. Boolean true
leader_cursor_max_dist Maximum distance away from the leader that their cursor can go to. Number 200
max_pikmin_in_field Maximum number of Pikmin that can be on the field. Number 100
throw_max_dist Maximum distance away from the leader that a throw can be aimed to. Number 200
whistle_growth_speed How quickly the whistle's radius grows in size, in pixels per second. Number 180
whistle_max_dist Maximum distance away from the leader that the whistle can focus on. Number 200
zoom_closest_reach At the closest zoom level, the camera shows an area depending on this value, in world pixels. You can think of this as "on average, the player has a horizontal view span or vertical view span of this amount". The area is calculated as this value squared.
Because different window resolutions can show more on one direction than another, and because the corners of the window show much farther than the sides, it's no use trying to control exactly how far the player sees, and so Pikifen instead focuses on the total area of visible world pixels at any moment. As an example, for an area of 87'025 pixels (295 squared), on a square window resolution like 1024x1024, the player will see 295 pixels far horizontally and 295 vertically. On a 16:9 resolution like 1024x768, the player will see around 340 horizontally and 256 vertically (around 87'040 pixels).		 Number 295
zoom_farthest_level Same as zoom_closest_reach, but for the farthest zoom level. Number 1340
