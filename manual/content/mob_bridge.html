Bridges are objects that start off as just a stub, and when Pikmin are done working on them, they form a full bridge over an obstacle that anything can walk on.

Examples of bridge types include the wooden bridge from Pikmin, that starts as a stub Pikmin work on and gets fully built when they are done, the red ceramic bridge from Pikmin 3, that starts as a stub and grows as Pikmin collect shards and drop them on the bridge, etc.

To create a new type of bridge, please follow the making object types tutorial.

Overview

The bridge object serves as something for the Pikmin to work on. As it gets damaged, it creates special objets of the built-in "Bridge component" type nearby, to serve as the floor and rails of the bridge proper. Once its health is fully depleted, it changes to a "destroyed" animation. Some bridges are built by having the Pikmin attack it (e.g. Pikmin wooden bridge). Others are built by having the Pikmin deliver some sort of resource into the bridge (e.g. Pikmin 3 ceramic bridge).

Properties

Common

  • All of the common object type properties.

Mandatory

main_texture Texture to use for the bridge's main floor. This texture must be in game_data/<pack>/graphics. Internal name
left_rail_texture Same as above, but the texture to use for the rail on the left side. Text
right_rail_texture Same as above, but the texture to use for the rail on the right side. Text

Others

rail_width Width for each rail. Number 16

Animations

A bridge type needs to have the following animations:

  • idling: Under construction, and is just idling.
  • destroyed: Fully built.

Script

Bridge types have hardcoded scripts, so the only thing you can do is enhance them.

Some script variables control special behavior, like the number of chunks the bridge is built into. To know which ones exist, you can go to the area editor, select a bridge, and read the tooltip of each variable.
