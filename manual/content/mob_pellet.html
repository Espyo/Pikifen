Pellets are carriable objects that simply give more Pikmin, when delivered to an Onion.

Examples of pellet types include the Red 1 pellet, the Red 5 pellet, the Yellow 1 pellet, etc.

To create a new type of pellet, please follow the making object types tutorial.

Overview

Each pellet object type can be bound to one type of Pikmin. Its color and look can be anything, though. That said, a pellet is forced to have a number drawn on top of it. Each pellet type is bound to one Pikmin type, and has data on how many seeds to give when the Pikmin type matches the Onion, and how many for when it doesn't.

Pellet type objects are hardcoded in the engine to be carriable, have a number displayed on top of them, have a number for Pikmin type match seeds, and a number for non-match seeds.

Properties

Common

  • All of the common object type properties.

Mandatory

number The number written on the pellet (e.g. 1, 5, 10). This will set the weight to the same thing. Number
number_image Image with the number. The image should be in game_data/<pack>/graphics. Internal name
pikmin_type Type of Pikmin the pellet matches with. Internal name

Recommended

match_seeds How many seeds it gives to the Onion when the pellet's Pikmin type matches the Onion's. Number 0
non_match_seeds How many seeds it gives to the Onion when the pellet's Pikmin type does not match the Onion's. Number 0

Animations

A pellet type needs to have the following animations:

  • idling: Idling; used for everything. This should not include the number, since the number is placed on top of the pellet during gameplay.

Script

Pellet types have hardcoded scripts, so the only thing you can do is enhance them.

Some script variables control special behavior, like the pellet's starting team. To know which ones exist, you can go to the area editor, select a pellet, and read the tooltip of each variable.
