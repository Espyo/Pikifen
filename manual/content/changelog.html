The following is a general changelog for Pikifen. This changelog will only list the most notable differences between each version of the engine; for a more detailed list of changes throughout the development period, check the commit log on GitHub.

Some versions may be unable to read content created in older versions. If you want to keep your content from one version to the next, read the "Breaking changes" list in that update's section.

Versions are numbered after the format major.minor.revision. The major number increases when the project is notably different from the major version before it. The minor number increases every time there are a bunch of changes. The revision number increases when there are bug fixes on the revision before it, and does not break compatibility unless that's necessary to fix something.

1.1.0

In development

Gameplay changes

  • Added logic for the fraction numbers to always show up if your cursor is over an object.

1.0.1

August 23rd, 2025

Gameplay changes

  • Fixed some pathing problems in Playful Yard's southern part.
  • Fixed the player being able to walk towards the southeastern treasure platform from the terrain north of it, in Olympian Ruins.
  • Fixed leaders released from inside other mobs not being able to be used. (Thanks Helodity)
  • Fixed buttons that open a folder or a webpage failing if the path had spaces in it. (Thanks Gulliblepikmin)
  • Fixed a crash when dismissing leaders with leaders, with the "dismiss all" option disabled. (Thanks Helodity, davdii)
  • Fixed a crash when an object starts a path on an area that has path stops, but none are accessible. (Thanks Helodity, Enderdude6)
  • Fixed Pikmin birth and death numbers in the pause menu not resetting between areas. (Thanks Gatling Groink)
  • Fixed Pikmin emitting "maturity down" particles when zapped or crushed.
  • Fixed children objects incorrectly counting as enemies for missions (e.g. each of the Beady Long Legs's feet). (Thanks Helodity)
  • Fixed Pikmin reaching treasures and corpses if they're near them horizontally, but far away vertically. (Thanks Helodity)
  • Fixed some screenshots in the help menu being outdated.
  • Fixed looping sounds in animations continuing to play even after leaving, in the animation editor. (Thanks Neo)
  • Fixed the OpenGL requirements being too high, now allowing the engine to run on weaker machines. (Thanks Helodity)

1.0.0

August 15th, 2025 – the "gold version" update

Gameplay changes

  • New audio:
    • Added a handful of songs. (Thanks Helodity, Rosey)
    • Added many new sounds.
  • New content:
    • Added new areas: Emerald Grove, Olympian Ruins, and Turbulent Tundra.
    • Added new enemies: Downy Snagret and Cyclopod.
    • Added new treasures: Chocolate Square and Chocolate Bar (internally a resource and pile respectively), Nugget and Nugget Pile (ditto), Cupcake, Soda Cap, and Yarn Ball.
  • New animations:
    • Added arms out, backflip, called, carrying light, carrying struggle, climbing, crushed, dying, flailing, knocked down dying, lounging, plucking thrown, pushing, shaking, sitting, sliding, and twirling animations to Pikmin.
    • Added called, looking around, sitting, stretching, and throwing animations to leaders.
    • Added Pikmin generation animations to Onions.
  • Quality of life:
    • Added the ability to customize the control binds for the system info and screenshots.
    • Added an option for how close a leader must be to an idle Pikmin before it's bumped.
    • Added an option that makes dismissing keep the current standby group, if there are multiple groups.
    • Added buttons for the options menu and statistics menu within the pause menu.
    • Added the ability to transfer 10 Pikmin at a time in the Onion menu.
    • Changed the standby type counter on the leader cursor to be controlled by an option, disabled by default.
    • Changed how carrying ties work in order to prioritize Pikmin types with a lower population and make the system more intuitive and predictable in general.
  • Balance:
    • Changed the effects of statuses like being on fire, poisoned, etc. Now, Pikmin lose one stage of maturity when they are afflicted.
    • Changed some properties and behaviors of some enemies to make them easier to fight.
  • Aesthetics:
    • Added more shadows to objects, including rectangular shadows. (Thanks Helodity)
    • Added icons to some menu buttons.
    • Changed the title screen Pikmin logo so that it represents all flower colors.
    • Changed the appearance of water to be more dynamic (using shaders). (Thanks Helodity)
    • Changed how window resolutions are handled. The same general amount of area pixels are shown regardless of resolution. i.e. at higher resolutions the player will see things sharper instead of seeing farther than normal.
    • Changed the score ruler in mission HUDs to look better. There is also now a "1 minute left!" notification on missions that are 2 minutes or longer.
  • Others:
    • Added the ability for missions to include the maker's score record.
    • Added Discord and GitHub buttons to the main menu.
    • Changed the way some things are calculated so the engine runs faster. The most extreme change is that objects that are off-camera and not close to a Pikmin/leader will not be processed.
    • Changed how options are organized in the options file, so you will likely have to set them again.
    • Changed the internal name of the enemy defeats statistic in the statistics file. Without external manipulation it will be impossible to port this value over, and it will be set to 0.

Content-making changes

  • Editors:
    • Added the particle editor. (Thanks Helodity)
    • Added a way to delete the current piece of content to all editors.
    • Added proper names to each entry of an editor's history.
    • Added a context menu (right-click) to most angle sliders, so you can pick East, South, West, North, quarter rotations, or 180 rotations quickly.
    • Changed the style of all editors to be less basic and more green.
  • Content organization:
    • Added generic properties to all pieces of content that have their data read from a data file, such as the maker name, version, notes, etc.
    • Added some sounds and songs to the system content names.
    • Added hard_bubble and frame_box to the system content names.
    • Added the game fonts to the system content names.
    • Added particle generators that used to be hardcoded to the system content.
  • Audio:
    • Added the tags property to songs.
    • Added the loop property to object type sound data.
  • Areas:
    • Added a "show blocking sectors" mode to the review panel of the area editor.
    • Added support for multiple sectors when you hold H to quickly set a sector's height in the area editor.
    • Added the ability for individual enemies to be considered bosses. (Thanks Helodity)
  • Animations:
    • Added a tint color property to sprites.
    • Added the ability for frames of animation to interpolate their transformation data (sprite translation, sprite rotation, etc.) from one to the next.
    • Added an option to set a custom background in the animation editor.
    • Added a "keep area" checkbox to the sprite transformation panel in the animation editor. This is useful for "squash and stretch" effects.
    • Added the ability to hold Shift while pressing the play/stop button in the animation editor in order to start from the beginning.
  • Scripts:
    • Added the stop_sound object script action.
    • Added the on_swallowed event.
    • Added a global block to scripts. Events in this block will have their actions run regardless of the current state.
    • Added an argument to the play_sound object script action for the sound ID.
    • Added an argument to the set_animation script action that makes the animation speed depend on the object's speed.
  • Maker tools:
    • Added the frame_advance and the set_song_pos_near_loop maker tools.
    • Added the ability to customize the control binds for maker tools.
    • Added some modifiers to the maker tools to allow for more behaviors.
  • Others:
    • Added the idle_bump_delay property and some radar color properties to the game configuration.
    • Added the knocked_down_duration and knocked_down_whistle_bonus properties to leader types.
    • Added the enemy_hit_rate_modifier_latched and enemy_hit_rate_modifier_standing properties to Pikmin types.
    • Added the carrying_light option to the Pikmin pose property of group task types.
    • Added the inactive_logic and use_damage_squash_and_stretch properties to object types.
    • Added the random_chance property to object type sounds.
    • Added the enemy_dead, leader_x, and obstacle icons to the radar's GUI definition.
    • Added messages, grown_resource and dropped_resource, for when a pile grows or drops a resource.
    • Added knockback multiplication logic to object type vulnerabilities. (Thanks Rosey)
    • Added the ability for enemies to revive after death. (Thanks Helodity)

Breaking changes

  • Content organization:
    • Changed the names and paths of all files and folders to be in snake case.
    • Changed how content refers to each other. Instead of using the content's proper name, it uses the "internal name", which is just the file's name without extension or the folder's name. For instance, the cardboard box's spawns block now refers to the flap object as cardboard_box_flap instead of Cardboard box flap.
    • Changed the folders game content is loaded from to use packs.
    • Changed the way content refers to assets. Now, content can refer to any graphics in the game_data/<pack>/graphics folders or any sounds in the game_data/<pack>/audio/sounds folders, and if needed, can include the subfolder.
  • Animations:
    • Changed the folder global animations are in. Instead of Animations (or animations), it's global_animations.
    • Changed the amount of hazards that a hitbox can have to 1. The property was thus renamed from hazards to hazard.
  • Areas:
    • Changed how mobs are saved inside of an area. Their block now specifies the internal name of the category, instead of the proper name. The type proper was also naturally changed to be the mob type's internal name.
    • Changed how sector types are saved inside of an area. Blocking sectors now use the internal name blocking instead of Blocking.
    • Changed the "maker notes" property for areas from notes to maker_notes.
    • Changed the names of area backup files so they match the names of normal area files. i.e. removed the _backup in their names.
    • Changed the amount of hazards that a sector can have to 1. The property was thus renamed from hazards to hazard.
    • Changed the names and values of enemy-killing things regarding area missions to be about enemy defeating instead, for the sake of accuracy. i.e. mission_fail_enemies_killed to mission_fail_enemies_defeated.
  • Audio:
    • Changed the song and track folders to be in the main audio folder instead of the music sub-folder.
    • Changed the title property in songs to instead be name. Because of the overall internal name change, the internal name that used to be in the name property is now the file name, meaning boss and menu music must be changed accordingly.
  • Scripts:
    • Changed the get_random_decimal script action to get_random_float.
    • Changed the focus and get_mob_info actions to support the same set of target object types.
  • Mob types:
    • Added the bounced_back animation to Pikmin types.
    • Changed the folder mob types are in. Instead of Types (or types), it's mob_types.
    • Changed the decimal type in an object type's area_editor_properties block to float.
    • Changed the names of the values of the type property for object type sounds.
    • Changed the property file to sound for object type sounds.
    • Changed the knocked_down leader animation to knocked_back.
    • Removed the drops_corpse enemy parameter. Now, enemies always drop a corpse when they finish dying. You can use the delete script action or avoid running finish_dying if you don't want a corpse.
    • Removed the is_boss enemy parameter. Now, individual enemies in an area can be marked as being bosses.
  • GUIs:
    • Changed the internal name of some menu-related GUI definitions for consistency: help menu, main menu's top page, all pause menus, and the control binds menu's GUI definitions.
    • Changed the names of the control-binds and sound effect related GUI items in the options menu.
    • Changed the enemy icon of the radar to enemy_alive.
    • Changed the files block of the radar and HUD GUIs to bitmaps.
    • Changed the name of the "screen" limit properties of the title screen GUI's logo block to read "window" instead.
  • Game configuration:
    • Changed the organization and some names of the game configuration properties.
    • Changed the zoom-level properties of the game configuration and options to instead be about the "reach", and renamed to zoom_*_reach accordingly.
    • Changed the idle Pikmin bump distance property of the game configuration by moving it to the options instead.
  • System content:
    • Changed where system content is loaded from. Instead of system_asset_file_names.txt, it's in system_content_names.txt. Animation names that used to be loaded from System_animations.txt are now also loaded from this new file.
    • Changed the system content name property for the title screen background from main_menu to title_screen_bg.
    • Changed enemy_spirit and pikmin_spirit to use "soul" instead of "spirit".
  • Base content:
    • Changed the location of graphics, sounds, and songs to be organized into sub-folders. Any use of any of these will need to be updated to point to the sub-folder and name (e.g. red_bulborb to enemies/red_bulborb).
    • Changed the bud_icon and flower_icon bitmaps to white_bud_icon and white_flower_icon.
    • Changed the white_gate bitmap to bramble_gate_white.
    • Changed the name of the White Spectralid to White Spectralids, along with its files.
    • Changed the internal name of some multi-word pieces of content to fit a subject_adjective naming scheme: electricity hazards, bridge-related objects, bramble gate-related objects, logic gate objects, Pellet Posy objects, pellet objects, Burgeoning Spiderwort objects, berry objects, crystal explosion particle generators, dust-related particle generators, gas-related particle generators, rock explosion particle generators, smoke-related particle generators, spark-related particle generators, speed-changing status effects, blackout weather conditions, and foggy weather conditions.
    • Changed the Stone_2.png texture to a JPG (stone_2.jpg).
    • Changed the internal name of Pikmin and enemy soul-related content to use "soul" instead of "spirit".
    • Changed how ceramic red bridges work so they no longer spawn a "base" child object. Now it has to be spawned manually if wanted. Also, renamed the base's image file from bridge_ceramic_red to bridge_base_ceramic_red.
  • Others:
    • Changed how particles work entirely.
    • Changed the properties of liquids.
    • Changed the affects_* properties of status effects to be a single affects property.
