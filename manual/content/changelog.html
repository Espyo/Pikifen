The following is a general changelog for Pikifen. This changelog will only list the most notable differences between each version of the engine; for a more detailed list of changes throughout the development period, check the commit log on GitHub.

Some versions may be unable to read content created in older versions. If you want to keep your content from one version to the next, read the "Content making changes" list in that update's section.

Versions are numbered after the format major.minor.revision. The major number increases when the project is radically different from the major version before it. The minor number increases every time there are a bunch of changes. The revision number increases when there are bug fixes on the revision before it, and does not break compatibility unless that's necessary to fix something.

0.26.0

In development

Gameplay changes

  • Added some songs.
  • Added many new sounds.
  • Added arms out, backflip, called, carrying struggle, climbing, crushed, dying, flailing, knocked down dying, lounging, plucking thrown, pushing, shaking, sitting, sliding, and twirling animations to Pikmin.
  • Added called, looking around, sitting, stretching, and throwing animations to leaders.
  • Added Pikmin generation animations to Onions.
  • Added the ability for missions to include the maker's score record.
  • Added new areas: Emerald Grove, Olympian Ruins, and Turbulent Tundra.
  • Added a new enemy: Cyclopod.
  • Added new treasures: Chocolate Square and Chocolate Bar (internally a resource and pile respectively), Soda Cap, Yarn Ball.
  • Added more shadows to objects, including rectangular shadows. (Thanks Helodity).
  • Added buttons for the options menu and statistics menu within the pause menu.
  • Added the ability to customize the control binds for the system info and screenshots.
  • Added an option for how close a leader must be to an idle Pikmin before it's' bumped.
  • Added the ability to transfer 10 Pikmin at a time in the Onion menu.
  • Changed the effects of statuses like being on fire, poisoned, etc. Now, Pikmin lose one stage of maturity when they are afflicted.
  • Changed the way some things are calculated so the engine runs faster. The most extreme change is that objects that are off-camera and not close to a Pikmin/leader will not be processed.
  • Changed the appearance of water to be more dynamic (using shaders). (Thanks Helodity)
  • Changed how options are organized in the options file, so you may have to set them again.
  • Changed some properties and behaviors of some enemies to make them easier to fight.
  • Changed how window resolutions are handled. The same general amount of area pixels are shown regardless of resolution. i.e. at higher resolutions the player will see things sharper instead of seeing farther than normal.
  • Changed the score ruler in mission HUDs to look better. There is also now a "1 minute left!" notification on missions that are 2 minutes or longer.
  • Changed the standby type counter on the leader cursor to be controlled by an option, disabled by default.
  • Changed how carrying ties work in order to prioritize Pikmin types with a lower population and make the system more intuitive and predictable in general.

Content-making changes

  • Changed the style of all editors to be less basic and more green.
  • Added a particle editor. (Thanks Helodity)
  • Content organization:
    • Changed from system asset names to system content names. The list now also includes more things that used to be hardcoded.
    • Added generic properties to all pieces of content that have their data read from a data file.
  • Sound features:

    • Added the set_song_pos_near_loop maker tool.
  • Animation features:

    • Added the ability for frames of animation to interpolate their transformation data (sprite translation, sprite rotation, etc.) from one to the next.
    • Added a "keep area" checkbox to the sprite transformation panel. This is useful for "squash and stretch" effects.
    • Added the ability to hold Shift while pressing the play/stop button in the animation editor in order to start from the beginning.
  • Added hard_bubble and frame_box to the system content names.
  • Added the idle_bump_delay property to the game configuration.
  • Added the tags property to songs.
  • Added the knocked_down_duration and knocked_down_whistle_bonus properties to leader types.
  • Added a second argument to the play_sound object script action for the sound ID.
  • Added the inactive_logic property to object types.
  • Added the stop_sound object script action.
  • Added icons to the menu GUIs.
  • Added Discord and GitHub buttons to the main menu GUI.
  • Added a "show blocking sectors" mode to the review panel of the area editor.
  • Added the game fonts to the system content names.
  • Added an option to set a custom background in the animation editor.
  • Added a global block to scripts. Events in this block will have their actions run regardless of the current state.
  • Added a parameter to the set_animation script action that makes the animation speed depend on the object's speed.
  • Added proper names to each entry of an editor's history.
  • Added support for mulitple sectors when you hold H to quickly set a sector's height.
  • Added an option to the hold_focused_mob action to allow the held object to forcefully be drawn above the holder.
  • Added the on_swallowed event.
  • Added a way to delete the current piece of content to all editors.
  • Added the enemy_dead, leader_x, and obstacle icons to the radar.
  • Added the ko animation to leaders.
  • Added radar color properties to the game config.
  • Added to the game content particle generators that used to be hardcoded.
  • Added the ability to customize the control binds for maker tools.
  • Added some modifiers to the maker tools to allow for more behaviors.
  • Added knockback multiplication logic to object type vulnerabilities. (Thanks Rosey)
  • Added the ability for enemies to revive after death. (Thanks Helodity)
  • Added a frame advance maker tool.
  • Added the enemy_hit_rate_modifier_latched and enemy_hit_rate_modifier_standing properties to Pikmin types.

Breaking changes

  • Content organization:
    • Changed the names and paths of all project files and folders to be in snake case.
    • Changed where game content is loaded from to use packs.
    • Changed how content refers to each other. Instead of using the content's proper name, it uses the "internal name", which is just the file's name without extension or the folder's name. For instance, the cardboard box's spawns block refers to the flap object as cardboard_box_flap instead of Cardboard box flap.
    • Changed how mobs are saved inside of an area. Their block now specifies the internal name of the category, instead of the proper name. The type proper was also naturally changed to be the mob type's internal name.
    • Changed how sector types are saved inside of an area. Blocking sectors now use the internal name blocking instead of Blocking.
    • Changed the way content refers to textures. Now, content can refer to any graphics in the game_data/<pack>/graphics folders, and if needed, can include the subfolder.
    • Changed the folder global animations are in. Instead of Animations (or animations), it's global_animations.
    • Changed the folder mob types are in. Instead of Types (or types), it's mob_types.
    • Changed the Stone_2.png texture to a JPG (stone_2.jpg).
    • Changed the names of area backup files so they match the names of normal area files. i.e. removed the _backup in their names.
    • Changed the "maker notes" property for areas from notes to maker_notes.
    • Changed the titleproperty in songs to instead be name. Because of the overall internal name change, the internal name that used to be in the name property is now the file name, meaning boss and menu music must be changed accordingly.
    • Changed where system content is loaded from. Instead of system_asset_file_names.txt, it's in system_content_names.txt. Animation names loaded from system_animations.txt are now also loaded from the new file.
    • Changed the location of graphics, sounds, and songs to be organized into sub-folders. Any use of any of these will need to be updated to point to the sub-folder and name (e.g. red_bulborb to enemies/red_bulborb).
    • Changed the song and track folders to be in the main audio folder instead of the music sub-folder.
  • Changed how particles work entirely.
  • Changed the knocked_down leader animation to knocked_back.
  • Changed the get_random_decimal script action to get_random_float.
  • Changed the decimal type in an object type's area_editor_properties block to float.
  • Changed the focus and get_mob_info actions to support the same set of target object types.
  • Changed the sound effect GUI elements in the options menu.
  • Changed the internal name of the pause menu and control binds menu GUIs from pause_help and control_binds_menu to help and options_menu_control_binds.
  • Changed the list of menu icons, adding an icon for the main menu's help button, for the particle editor, and for the pack management menu.
  • Changed the affects_* properties of status effects to be a single affects property.
  • Changed the bud_icon and flower_icon bitmaps to white_bud_icon and white_flower_icon.
  • Changed the properties of liquids.
  • Changed the name of the White Spectralid to White Spectralids, along with its files.
  • Changed the enemy icon of the radar to enemy_alive.
  • Changed the names of the editor background texture and sound effect volume options.
  • Changed the names of the values of the type property for object type sounds.
  • Changed the system content name property for the title screen background from main_menu to title_screen_bg.
  • Changed the organization and some names of the game configuration properties.
  • Changed the names of the control-binds related GUI items in the options menu.
  • Changed the amount of hazards that a sector or a hitbox can have to 1. Their properties were thus renamed from hazards to hazard.
  • Changed the name of the "screen" limit properties of the title screen GUI's logo block to read "window" instead.
  • Changed the zoom-level properties of the game configuration and options to instead be about the "reach", and renamed to zoom_*_reach accordingly.
  • Changed the idle Pikmin bump distance property of the game configuration by moving it to the options instead.
  • Changed the names and values of enemy-killing things regarding area missions to be about enemy defeating instead, for the sake of accuracy. i.e. mission_fail_enemies_killed to mission_fail_enemies_defeated
  • Changed the property enemy_deaths in the statistics to enemy_defeats. Without external manipulation it will be impossible to port this value of the statistics file over to this version.
  • Changed the internal name of some multi-word pieces of content to fit a "subject_adjective" naming scheme: electricity hazards, bridge-related objects, bramble gate-related objects, logic gate objects, Pellet Posy objects, pellet objects, Burgeoning Spiderwort objects, berry objects, crystal explosion particle generators, dust-related particle generators, gas-related particle generators, rock explosion particle generators, smoke-related particle generators, spark-related particle generators, speed-changing status effects, blackout weather conditions, and foggy weather conditions.
  • Changed the internal name of some menu-related GUI definitions for consistency: help menu, main menu's top page, and all pause menu GUI definitions.
  • Changed the internal name of Pikmin and enemy soul-related content to use "soul" instead of "spirit".
  • Removed the drops_corpse enemy parameter. Instead, use the delete script action or avoid running finish_dying.

0.25.0

February 17th, 2024 – the "music & radar" update

Gameplay changes

  • New pause menu features:
    • Added a radar to the pause menu, along with Go Here, and a dedicated button shortcut.
    • Added a Pikmin status page to the pause menu.
  • New objects:
    • Added levers and timed levers. Interact with a leader or throw a Pikmin to power or stop powering devices.
    • Added buttons. Interact with a leader or throw a Pikmin to power devices for a split-second.
    • Added logic gates. These allow for complex mechanisms, like a gate that only opens if two different power sources are on.
  • Audio improvements:
    • Added audio volume options.
    • Changed the audio system for much better sound effect playback.
  • UI improvements:
    • Added headers to each menu.
    • Added the ability to close messages with the "pause" button.
    • Added control binds for navigating between pages of the pause menu, as well as radar-related binds.
    • Changed the options menu to be split into pages.
  • Other changes:
    • Changed how some objects push against each other. Now, leaders and Pikmin can squeeze through them to get unstuck. This does not apply to more "solid" objects like gates.
    • Changed the range for an idle Pikmin to be considered "bumped" by the active leader, so they can be called from farther away.
    • Changed the way the Yellow Wollywog works to make the fight easier and a bit more dynamic.

Content making changes

  • Audio:
    • Added music support.
    • Added the system sound effects to the system asset file names list.
    • Added the ability for objects to play sounds via animations or scripting (play_sound action).
    • Added the ability for different Pikmin types to play different sounds.
  • Area editor improvements:
    • Added a redo feature to the area editor.
    • Added the ability for a sector to reuse the same vertex. Before, it just considered the sector broken.
    • Added the ability to create sub-sectors. This happens if one sector's polygon ends up getting split into two. While not recommended, it stops the maker from receiving warnings and from the sector to be considered broken.
    • Added the ability to keep drawing a new sector, for when you try to split a sector but the engine considers that split useless.
    • Added an outline of child objects when viewing an object in the area editor. (Thanks Helodity)
  • New object type properties:
    • Added the pushes_softly property.
    • Added the blackout_radius property. (Thanks Helodity)
    • Added the area_editor_recommend_links_from and area_editor_recommend_links_to properties.
  • Object script event and action changes:
    • Added the on_input_received event.
    • Added the on_ready event.
    • Added the get_area_info, get_event_info, and get_mob_info actions. All data obtained from get_info are now obtained by the corresponding actions. (Thanks Helodity)
    • Removed the get_info action, since it was replaced by the previous three actions. (Thanks Helodity)
    • Removed the get_focus_info action, since it was replaced by get_mob_info with the focus target. (Thanks Helodity)
    • Added the input_name and input_value properties to the get_event_info action.
    • Added the id and state data types to the get_mob_info action. (Thanks Helodity)
  • New game configuration properties:
    • Added support for the thus far-unused maturity_power_mult property.
    • Added the idle_bump_range property.
  • Pathing changes:
    • Added the "ledge" link type. Normal mobs can go from a higher path stop to a lower one but not the other way around, unless they are airborne.
    • Added the "script use only", "light load only", and "airborne only" settings to stops. These replace the equivalent link types. Also, added labels to stops, which replace labels in links.
    • Removed all link types except for the newly-added "ledge" type, since they were replaced by the equivalent stop settings. Also, removed link labels, since they too were replaced with a stop property.
    • Added a radius property to path stops.
  • Other changes:
    • Added three new player actions to the controls: custom_a, custom_b, and custom_c. These do nothing on their own, but can be detected through leader scripts. (Thanks WanderingLetterS)
    • Added the is_boss property to enemy types.
    • Added the radar_color property to liquids.

Breaking changes

  • Removed the get_info mob action, per above.
  • Removed the get_focus_info mob action, per above.
  • Changed the GUI data and file names for the options menu since it's now split into pages, as well as the name of the control binds menu (Control_binds_menu.txt instead of Controls_menu.txt).
  • Changed the list of existing system asset file names to be inside of a graphics block.
  • Changed the location of sound effects from Game_data/Audio to Game_data/Audio/Sounds.
  • Changed the way leader type and Pikmin type sounds are configured. Instead of properties starting with sfx_, or just hardcoded files starting with Pikmin_, they are now standard object type sounds.

0.24.0

July 14th, 2023 – the "new areas" update

Gameplay changes

  • New areas:
    • Added Tutorial Meadow. If you never played an area before, you will be prompted to start here.
    • Added Cryptic Atrium. A medium-short puzzle stage with pressure plates and automatic gates.
    • Added Playful Yard. A reincarnation of the old "Play" area, with a more playful motif, better design, and as a proper mission.
    • Added Crash Landing. A difficult quick mission to get all treasures before time runs out. (Thanks Helodity)
    • Added DEV Lab. It showcases pretty much everything Pikifen has to offer.
    • Removed Play in favor of Playful Yard.
  • New objects:
    • Added orb signs, which work like signs but are generic orbs instead of wooden planks.
    • Added geysers, a type of bouncer that only leaders can use.
    • Added carriable rocks, which can be carried around and dropped to weigh on something.
    • Added pressure plates, which trigger some other object to do something whenever they have enough weight on them.
    • Added automatic gates, which lift in the air whenever they are powered by things like a pressure plate.
    • Added carry targets, which allows other things to be carried to it for no effect.
    • Added some treasures: a fidget spinner, a yo-yo, and a jigsaw puzzle piece.
  • Misc:
    • Added images to the help page of the pause menu. (Thanks erdinvark)
    • Added a random area button to the area selection menu.
    • Added mission statistics for the total of clears, total of platinum medals, and total combined sum of score points.
    • Changed the way paper bags work to make it so leaders can't be thrown on top, and so that Pikmin can't accidentally be thrown over them to the other side.

Content making changes

  • Editor improvements:
    • Added a bunch of menu bar items across all editors.
    • Added a toolbar button to the area editor to toggle preview mode. Advanced interface mode only.
    • Added the ability to lock to an axis while dragging, if Ctrl is held.
    • Added the ability to add to the selection when dragging a selection box, if Ctrl is held.
    • Added the ability to copy a sector, edge, object, or path link's properties, so they can then be pasted elsewhere.
    • Added some shortcut operations to the sector height widget as a right-click context menu.
    • Added settings for the area editor's path preview tool. Plus some more detailed information about the path calculation.
    • Changed the unsaved changes warning to be a dialog like in most programs. The idea of the old pop-up was to require fewer clicks and mouse movement when you were sure you wanted to not save, but some actions are hidden behind menus which forces you to go there twice. The dialog approach comes with many quality-of-life tweaks that make it an overall better feature.
    • Changed the way copying textures from a different sector works. Instead of selecting the source, then hovering the mouse over the destination and pressing T, you now copy the source sector's properties, then paste just the texture on the selected sectors.
    • Changed what happens when the user is drawing a new sector and clicks on an edge or vertex. Instead of throwing an error, the editor now changes the drawing to a sector split drawing, and lets the user continue drawing from the opposite end.
    • Changed the area editor's saving procedure to no longer need to boot the user back to the main panel.
  • Others:
    • Added the hud maker tool.
    • Added the path_info maker tool.
    • Added the custom_carry_spots property to mob types.
    • Added the power_goal script variable to group tasks.
    • Added the goal_number script variable to scales.
    • Added the stay value to a resource type's delivery results.
    • Added the linked_mob_matching_type value to a resource type's carrying destinations.
    • Added the area_image_padding property to the maker tools.
    • Added the ability for dead enemies to drop objects via a "stored inside" feature in the area editor. This allows the engine to detect things like treasures inside enemies for the purposes of missions.

Breaking changes

  • Removed the ability for dead enemies to drop objects via the spoils and random_pellet_spoils script variables, in favor of the "stored inside" area editor feature. Areas with such objects need to be adjusted.
  • Changed the following obstacles to have a consistent 192px size: electrical wire (also now doesn't push), paper bag (also now blocks paths), cardboard box, wood bridge.
  • Changed the spritesheet for player input icons (formerly control icons). All properties and files changed from control_icon to player_input_icon.
  • Changed the show_hud_controls option to show_hud_input_icons.
  • Changed the gameplay GUI item names that had button in them to instead have input.
  • Changed the default value of an object type's can_walk_on_others properties to false.
  • Changed the name of some texture files: Bark → Wood_bark. Bridge → Wood_bridge. Bridge_rail → Wood_bridge_rail. Metal → Metal_rusted. Stump → Wood_stump.
