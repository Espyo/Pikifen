The following is a general changelog for Pikifen. This changelog will only list the most notable differences between each version of the engine; for a more detailed list of changes throughout the development period, check the commit log on GitHub.

Some versions may be unable to read content created in older versions. If you want to keep your content from one version to the next, read the "Breaking changes" list in that update's section.

Versions are numbered after the format major.minor.revision. The major number increases when the project is notably different from the major version before it. The minor number increases every time there are a bunch of changes. The revision number increases when there are bug fixes on the revision before it, and does not break compatibility unless that's necessary to fix something.

1.1.1

December 29th, 2025

Gameplay changes

  • Fixed some objects being carried to Onions instead of their correct destination. (Thanks Helodity)

1.1.0

December 25th, 2025 – the "I/O Polish" update

Gameplay changes

  • Inventory:
    • Added an inventory system to replace the old spray management system. It includes an inventory that can be opened or closed, as well as shortcuts that can be pressed to use certain items directly without the inventory menu. The inventory shows spray concoction progress, and can be controlled in the options to work like a normal menu or a quick selector.
    • Changed the "lie down" action to instead be an item in the inventory, the Napsack. It now also no longer allows Pikmin to carry the leader, since that functionality is better done via Go Here!, and instead heals the leader slowly.
  • Better controls:
    • Added proper support for Nintendo Switch Pro Controllers and Microsoft X-Box 360 (and Steam Deck) controllers. Button icons will now show the proper name or image for recognized buttons and sticks.
    • Changed gamepad logic to be less buggy and more intuitive.
    • Changed non-mouse menu navigation logic to be more intuitive. Holding the input for a direction now also navigates continuously.
    • Fixed the leader cursor getting stuck as well as being sluggish, when controlling it outside of a mouse.
  • GUI improvements:
    • Added logic for the fraction numbers to always show up, as well as for the enemy/treasure points to display, if the leader's cursor is over an applicable object. (Thanks Tenacious)
    • Added a shaking effect to the leader's health wheel when said leader gets hurt.
    • Added indicators for mission exit regions and for mission-critical objects to the radar. The former also shows up in-game.
    • Added a swirling effect to the background of the Onion menu. (Thanks Helodity)
    • Changed how the Onion menu works so it's easier to navigate without the mouse, and so that when there are too many Pikmin types, the list can be scrolled through smoothly instead of in concrete pages.
    • Changed the way difficulty is shown for an area in the area menu to use more subjective and stylish flame icons, instead of overly-specific text.
  • New options:
    • Added option to allow the mouse cursor to move the leader.
    • Added option to make the mouse cursor automatically hide when it hasn't been moved in a while.
    • Added option that pauses gameplay if the game window loses focus.
    • Added option for a shader compatibility mode. If you experience crashes with shaders, turning this on might stop the crashes at the cost of some graphical effects going missing altogether.
  • System:
    • Added system notifications. A small pop-up will appear when a controller is connected or disconnected, as well as when the options are saved.
    • Changed the system info feature to work in menus and editors, not just the gameplay state. It now also shows more version information.
  • Others:
    • Added a new enemy: Plasm Wraith. (Thanks Helodity)
    • Added camera shake to some things, like bomb rock explosions or the Beady Long Legs falling. Its strength can be controlled in the options.
    • Added thrown, KO'd, climbing, and sliding animations to leaders. The KO animation existed already, but is now completely new, with the leader falling stomach-down.
    • Added the ability for leaders to have glowing antenna lights, with a trail of particles behind them.

Content-making changes

  • GUI:
    • Added a way to create custom GUI items. Some of the existing hardcoded GUI items have also been made into custom items.
    • Added descriptions to the hardcoded GUI items, visible in the item list in the GUI editor. The full lists have been removed from the manual page as a result.
    • Added some GUI colors to the game configuration.
    • Added a modal dialog GUI definition.
  • Maker tools:
    • Added an option that makes it so using a maker tool in normal gameplay throws a warning, and a confirmation for if the player really wants to use it. This helps with accidental activations of maker tools during what should be normal gameplay.
    • Added the "show reaches" maker tool, which shows a mob's far and near reaches.
  • Script actions:
    • Added the else_if, set_focus_var, and shake_camera actions.
    • Added the absolute_number, ceil_number, ease_number, floor_number, get_angle_clockwise_difference, get_angle_smallest_difference, interpolate_number, round_number, and square_number mathematics script actions.
    • Added the ^ (power) operator to the calculate action.
    • Changed the print action to show the latest and the past lines, instead of just the latest.
  • Editor improvements:
    • Added a quick play feature to all editors.
    • Added an "Open externally" menu item to all editors, which opens the content's file/folder in your operative system.
    • Added animation editor menus to quickly jump from a sprite to the animations that use it, and also from an animation's frame to the sprite it uses.
    • Added an "Apply changes to all matching sprites" button to the animation editor's sprite bitmap panel.
    • Added a tool to the animation editor that flips all hitboxes of a sprite vertically or horizontally.
    • Added a tool to the animation editor that makes all hitboxes of a sprite vertically or horizontally symmetrical.
    • Added some edge sector patching tools to the area editor. Useful for when you have sectors broken beyond normal repair.
  • Others:
    • Added the arrow_*, difficulty, mission_mob, and warning system content images.
    • Added a buildup mechanic to statuses.
    • Added the ingredient_name property to spray types.
    • Added the vertical_speed_multiplier property to mob types.

Compatibility changes

  • GUI:
    • Changed some GUI items that used to be hardcoded to instead be custom items.
    • Changed the way hardcoded GUI items are stored in GUI definitions. Instead of one property per item in a positions block, each item is a sub-block inside of a hardcoded_items block.
    • Changed the Onion menu GUI definition, in accordance to the new layout. The definitions got split into two.
    • Changed how some prompts work so that they use the new modal dialog GUI definition instead of their own definitions. The main menu tutorial question and pause menu confirmation question GUI definitions were removed.
    • Deprecated the cursor_speed, cursor_cam_weight and show_counter_on_cursor options GUI items. Please rename them to leader_cursor_speed, leader_cursor_cam_weight and show_leader_cursor_counter.
  • Base content assets:
    • Changed the cursor.png, cursor_counter.png, and notification.png images to leader_cursor.png, leader_cursor_counter.png, and leader_prompt.png.
    • Changed the menu_selected.ogg sound to menu_focus.ogg.
    • Changed the player_input_icons.png image so direction-based icons follow a right-down-left-up pattern, so it's consistent across the project, and added a handful of icons.
  • Others:
    • Changed the status effect type property glow to colorize. Instead of a pseudo-additive color effect, it now changes the mob's pixel colors towards the target color, with the strength specified by the alpha component.
    • Changed the leader animation lying to sleeping.
    • Deprecated the options cursor_speed, cursor_cam_weight, show_counter_on_cursor, and advanced > p*_mouse_moves_cursor. Play 1.1.0 to automatically rename them to the new names leader_cursor_speed, leader_cursor_cam_weight, show_leader_cursor_counter, and controls > p*_mouse_moves_leader_cursor.
    • Deprecated the system content properties menu_selected, notification, cursor, and cursor_counter. Please rename them to menu_focused, leader_prompt, leader_cursor, and leader_cursor_counter.
    • Deprecated the game configurations cursor_spin_speed and cursor_max_dist. Please rename them to mouse_cursor_spin_speed and leader_cursor_max_dist.
