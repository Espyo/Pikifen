0.25.0

February 17th, 2024 – the "music & radar" update

Gameplay changes

  • New pause menu features:
    • Added a radar to the pause menu, along with Go Here, and a dedicated button shortcut.
    • Added a Pikmin status page to the pause menu.
  • New objects:
    • Added levers and timed levers. Interact with a leader or throw a Pikmin to power or stop powering devices.
    • Added buttons. Interact with a leader or throw a Pikmin to power devices for a split-second.
    • Added logic gates. These allow for complex mechanisms, like a gate that only opens if two different power sources are on.
  • Audio improvements:
    • Added audio volume options.
    • Changed the audio system for much better sound effect playback.
  • UI improvements:
    • Added headers to each menu.
    • Added the ability to close messages with the "pause" button.
    • Added control binds for navigating between pages of the pause menu, as well as radar-related binds.
    • Changed the options menu to be split into pages.
  • Other changes:
    • Changed how some objects push against each other. Now, leaders and Pikmin can squeeze through them to get unstuck. This does not apply to more "solid" objects like gates.
    • Changed the range for an idle Pikmin to be considered "bumped" by the active leader, so they can be called from farther away.
    • Changed the way the Yellow Wollywog works to make the fight easier and a bit more dynamic.

Content making changes

  • Audio:
    • Added music support.
    • Added the system sound effects to the system asset file names list.
    • Added the ability for objects to play sounds via animations or scripting (play_sound action).
    • Added the ability for different Pikmin types to play different sounds.
  • Area editor improvements:
    • Added a redo feature to the area editor.
    • Added the ability for a sector to reuse the same vertex. Before, it just considered the sector broken.
    • Added the ability to create sub-sectors. This happens if one sector's polygon ends up getting split into two. While not recommended, it stops the maker from receiving warnings and from the sector to be considered broken.
    • Added the ability to keep drawing a new sector, for when you try to split a sector but the engine considers that split useless.
    • Added an outline of child objects when viewing an object in the area editor. (Thanks Helodity)
  • New object type properties:
    • Added the pushes_softly property.
    • Added the blackout_radius property. (Thanks Helodity)
    • Added the area_editor_recommend_links_from and area_editor_recommend_links_to properties.
  • Object script event and action changes:
    • Added the on_input_received event.
    • Added the on_ready event.
    • Added the get_area_info, get_event_info, and get_mob_info actions. All data obtained from get_info are now obtained by the corresponding actions. (Thanks Helodity)
    • Removed the get_info action, since it was replaced by the previous three actions. (Thanks Helodity)
    • Removed the get_focus_info action, since it was replaced by get_mob_info with the focus target. (Thanks Helodity)
    • Added the input_name and input_value properties to the get_event_info action.
    • Added the id and state data types to the get_mob_info action. (Thanks Helodity)
  • New game configuration properties:
    • Added support for the thus far-unused maturity_power_mult property.
    • Added the idle_bump_range property.
  • Pathing changes:
    • Added the "ledge" link type. Normal mobs can go from a higher path stop to a lower one but not the other way around, unless they are airborne.
    • Added the "script use only", "light load only", and "airborne only" settings to stops. These replace the equivalent link types. Also, added labels to stops, which replace labels in links.
    • Removed all link types except for the newly-added "ledge" type, since they were replaced by the equivalent stop settings. Also, removed link labels, since they too were replaced with a stop property.
    • Added a radius property to path stops.
  • Other changes:
    • Added three new player actions to the controls: custom_a, custom_b, and custom_c. These do nothing on their own, but can be detected through leader scripts. (Thanks WanderingLetterS)
    • Added the is_boss property to enemy types.
    • Added the radar_color property to liquids.

Breaking changes

  • Removed the get_info mob action, per above.
  • Removed the get_focus_info mob action, per above.
  • Changed the GUI data and file names for the options menu since it's now split into pages, as well as the name of the control binds menu (Control_binds_menu.txt instead of Controls_menu.txt).
  • Changed the list of existing system asset file names to be inside of a graphics block.
  • Changed the location of sound effects from Game_data/Audio to Game_data/Audio/Sounds.
  • Changed the way leader type and Pikmin type sounds are configured. Instead of properties starting with sfx_, or just hardcoded files starting with Pikmin_, they are now standard object type sounds.

0.24.0

July 14th, 2023 – the "new areas" update

Gameplay changes

  • New areas:
    • Added Tutorial Meadow. If you never played an area before, you will be prompted to start here.
    • Added Cryptic Atrium. A medium-short puzzle stage with pressure plates and automatic gates.
    • Added Playful Yard. A reincarnation of the old "Play" area, with a more playful motif, better design, and as a proper mission.
    • Added Crash Landing. A difficult quick mission to get all treasures before time runs out. (Thanks Helodity)
    • Added DEV Lab. It showcases pretty much everything Pikifen has to offer.
    • Removed Play in favor of Playful Yard.
  • New objects:
    • Added orb signs, which work like signs but are generic orbs instead of wooden planks.
    • Added geysers, a type of bouncer that only leaders can use.
    • Added carriable rocks, which can be carried around and dropped to weigh on something.
    • Added pressure plates, which trigger some other object to do something whenever they have enough weight on them.
    • Added automatic gates, which lift in the air whenever they are powered by things like a pressure plate.
    • Added carry targets, which allows other things to be carried to it for no effect.
    • Added some treasures: a fidget spinner, a yo-yo, and a jigsaw puzzle piece.
  • Misc:
    • Added images to the help page of the pause menu. (Thanks erdinvark)
    • Added a random area button to the area selection menu.
    • Added mission statistics for the total of clears, total of platinum medals, and total combined sum of score points.
    • Changed the way paper bags work to make it so leaders can't be thrown on top, and so that Pikmin can't accidentally be thrown over them to the other side.

Content making changes

  • Editor improvements:
    • Added a bunch of menu bar items across all editors.
    • Added a toolbar button to the area editor to toggle preview mode. Advanced interface mode only.
    • Added the ability to lock to an axis while dragging, if Ctrl is held.
    • Added the ability to add to the selection when dragging a selection box, if Ctrl is held.
    • Added the ability to copy a sector, edge, object, or path link's properties, so they can then be pasted elsewhere.
    • Added some shortcut operations to the sector height widget as a right-click context menu.
    • Added settings for the area editor's path preview tool. Plus some more detailed information about the path calculation.
    • Changed the unsaved changes warning to be a dialog like in most programs. The idea of the old pop-up was to require fewer clicks and mouse movement when you were sure you wanted to not save, but some actions are hidden behind menus which forces you to go there twice. The dialog approach comes with many quality-of-life tweaks that make it an overall better feature.
    • Changed the way copying textures from a different sector works. Instead of selecting the source, then hovering the mouse over the destination and pressing T, you now copy the source sector's properties, then paste just the texture on the selected sectors.
    • Changed what happens when the user is drawing a new sector and clicks on an edge or vertex. Instead of throwing an error, the editor now changes the drawing to a sector split drawing, and lets the user continue drawing from the opposite end.
    • Changed the area editor's saving procedure to no longer need to boot the user back to the main panel.
  • Others:
    • Added the hud maker tool.
    • Added the path_info maker tool.
    • Added the custom_carry_spots property to mob types.
    • Added the power_goal script variable to group tasks.
    • Added the goal_number script variable to scales.
    • Added the stay value to a resource type's delivery results.
    • Added the linked_mob_matching_type value to a resource type's carrying destinations.
    • Added the area_image_padding property to the maker tools.
    • Added the ability for dead enemies to drop objects via a "stored inside" feature in the area editor. This allows the engine to detect things like treasures inside enemies for the purposes of missions.

Breaking changes

  • Removed the ability for dead enemies to drop objects via the spoils and random_pellet_spoils script variables, in favor of the "stored inside" area editor feature. Areas with such objects need to be adjusted.
  • Changed the following obstacles to have a consistent 192px size: electrical wire (also now doesn't push), paper bag (also now blocks paths), cardboard box, wood bridge.
  • Changed the spritesheet for player input icons (formerly control icons). All properties and files changed from control_icon to player_input_icon.
  • Changed the show_hud_controls option to show_hud_input_icons.
  • Changed the gameplay GUI item names that had button in them to instead have input.
  • Changed the default value of an object type's can_walk_on_others properties to false.
  • Changed the name of some texture files: Bark → Wood_bark. Bridge → Wood_bridge. Bridge_rail → Wood_bridge_rail. Metal → Metal_rusted. Stump → Wood_stump.

0.22.0

July 5th, 2022 – the "polish" update

Gameplay changes

  • Added a help page to the pause menu, with gameplay information, tips, controls, Pikmin type explanations, and more.
  • White Pikmin poison now applies a status effect that drains health over time. Enemies still end up losing the same amount of health in the end.
  • Pikmin are now better at detecting gates and other rectangular objects.
  • Pikmin can now circle around enemies when in combat.
  • Bridges now grow in chunks, and the point Pikmin deliver fragments to, or the position of the pile of sticks, now moves along with the bridge.
  • Misc. graphical enhancements:
    • All gates have received graphical improvements.
    • Decorative objects no longer cast light in blackout weather.
    • When Pikmin deliver an object to an Onion, ship, etc., the sucking animation is now more dynamic. Also, added a tossing animation for delivering bridge fragments to bridges.
    • Improved the look of the ship's beam and the leader's whistle. The pretty_whistle option has been removed. (Thanks Helodity)
    • HUD elements now animate when things change, and slide off-frame when menus are entered in-game. (Thanks Helodity)
    • In-game message boxes are now much more animated.
    • Health wheels and fraction numbers now appear and disappear smoothly, and are more animated in general.
    • Enemies now squash and stretch when damaged.
    • Objects being carried now sway.
    • Icons for controls have been improved.
  • Fixed leaders being able to grab group members from the other side of walls or objects with collision, like gates.

Content making changes

  • Bridges are now made up of objects. This should make the process of placing bridges in an area much simpler, since there's no need to fiddle around with sectors. Areas with bridges need to be updated to use this new bridge format.
  • Status effects now have a health_change property, which controls how much health the object loses per second in an absolute value, instead of it being a ratio of total health.
  • Spike damage types now have a status_to_apply property, which allows the damaged object to receive a status effect.
  • Object types now have a status_vulnerabilities block, which works like the other vulnerability blocks, but controls damage taken from status effects.
  • Object types now have a terrain_radius property, which controls their collision against walls.
  • Object types now have a description property, which can be used to briefly describe it. This appears in the area editor, and Pikmin types make use of this for the help page of the in-game pause menu.
  • The tooltip of an object type's area editor property now word-wraps automatically. If you relied on line breaks (\n) to do this before, you may want to adjust it.
  • The set_animation script action now accepts the options random_time and random_time_on_spawn, to let the animation start on a random time, or start on a random time but only if the mob just spawned.
  • Added the on_receive_delivery object script event.
  • Added the health_ratio option to the get_info object script action.
  • Added the health script variable.
  • Added the max_health script variable.
  • Added the bright_circle and bright_ring system assets.
  • The area editor now also shows the terrain radius of selected objects (if the "Show territory/terrain radius" option is on).
  • Converters now use an animation called bumped instead of bumping, for consistency's sake.
  • System assets no longer have icons for mouse-related controls, but instead have one spreadsheet for many special control icons.
  • The gameplay HUD received all sorts of new items, and some have been renamed to be more consistent.
  • Areas and animations now save the engine version they were edited in. This will likely be used in the future to allow the engine to automatically port older content to newer versions.

0.21.0

October 22nd, 2021 – the "Pikmin 3" update

Gameplay changes

  • Added Rock Pikmin.
    • When thrown at an enemy, they cause damage when they collide, and then bounce back.
    • They also lunge forward on their own when they get close to an enemy they are chasing on foot. This impact works just like a throw, bounce back included.
    • Also, updated the pellets, Pellet Posies, Candypop Buds, and Onions to support Rock Pikmin.
    • Plus, added crystal walls, crystal nodules, large crystals, small crystals, and Skutterchucks.
  • Added Winged Pikmin.
    • If their target is above or below them, they can move up and down in the air to get to it.
    • When following a leader, they try to stay a bit above the leader, and when idling, they also try to idle some distance above-ground.
    • Because they are flying in the air when following a leader, they can travel over bodies of water. That said, leaders that are underwater can't grab or throw them.
    • Also, updated the pellets, Pellet Posies, Candypop Buds, and Onions to support Winged Pikmin.
    • Plus, added bamboo gates, Flukeweeds, and Puffy Blowhogs.
  • Added Bulbmin, both adults and young. They are invulnerable to all elemental hazards, but the young lack an Onion or ship to go to.
  • Added Alph, Brittany, and Charlie, as well as the S.S. Drake. The Hocotatian leaders also got a glow-up. The President's color was changed to yellow, otherwise half of the leaders would be blue.
  • Purple Pikmin now cause a bit of collision damage when they land on an enemy after being thrown.
  • Enemies petrified with the ultra-bitter spray now shake before breaking free, their petrification visually bursts when it's over, and their animations now properly freeze.
  • Munge Dweevils can now pick up treasure.
  • Pikmin will now only take paths that they can survive. e.g. a carcass with 1 Red Pikmin and 9 Blue Pikmin carrying it will take a path that doesn't go through water, but a carcass carried by 10 Blue Pikmin might take that water path.
  • Objects now move with acceleration. This should make them a bit smoother when moving.
  • Revamped the cursor.
    • The leader cursor's color now matches the standby Pikmin's color. If there is none, it uses a faded cyan color.
    • The number of Pikmin of the standby type now also appears next to the cursor.
    • There is now a line connecting the leader to the cursor. When the leader has a group member, the line changes to represent a preview of how that member would be thrown. This helps to show which ledges can and can't be reached.
    • Added an option to set the cursor's speed, used when controlling it without a mouse. (Thanks MonkeyUhOh)
  • If a Pikmin is lying down, a leader can now whistle it to make it get up faster, and return to the leader once it does.
  • Added an option to allow the leader to auto-throw. If you prefer not to mash the throw button, consider turning this on.
  • If a new leader shows up from a defeated enemy, their HUD information will now display properly. (Thanks pikminjosh)
  • All menus now have a tooltip for the currently selected item.
  • The in-game message box now looks like other boxes, instead of a stretched flat image.
  • Fullscreen mode now uses a simple borderless window. True fullscreen mode can still be enabled in the options file, but comes with risks.

Content making changes

  • Rock Pikmin-related features:
    • Added a property for Pikmin types, attack_method, that controls how the Pikmin attacks. If set to impact, that Pikmin attacks like a Rock Pikmin, as explained above.
    • Their on-foot lunges use the attack animation, so it should ideally be in a pose where they lunge forward, and have a hitbox.
  • Winged Pikmin-related features:
    • Added a property for Pikmin types, can_fly, that makes a Pikmin fly like a Winged Pikmin, as explained above.
    • These Pikmin can only move vertically if the state they are in allows it (e.g. they can't fly if they're being knocked back).
    • Technically, if a leader is on a floor with a hazard (liquid or otherwise), they will be unable to grab Pikmin that are outside it and are vulnerable to the hazard. If they grab and bring those Pikmin to said floor, the Pikmin will suffer the effects.
  • Pikmin now have a new state that is entered when they land on a mob, mob_landing. In it, they change to an animation of the same name. It is only after this animation ends that they latch on or bounce back. As such, this animation is a good opportunity to get the Pikmin to cause damage upon collision.
  • Revamped paths.
    • If a path stop is on top of a sector with hazards, objects will now be unable to traverse it if they are vulnerable to the hazards. In the case of a carried object, this applies to the carrier Pikmin. If there is another path, they will take it, otherwise they will stay stuck in front of that path stop. If the hazard is only on the floor, and the object or carrier Pikmin can fly, then they will still take that path.
    • Path links now have a type. "Normal" works like normal, "custom only" can only be used by objects that need that path for scripting, "light load only" can only be taken by objects carrying a weight of 1 or nothing, and "airborne only" can only be taken by flying objects.
  • Revamped the status system.
    • Added the animation_change property, which lets the object's animation be changed when the status is applied.
    • Added the freezes_animation property. If true, the object will stay stuck in its current sprite until the status effect is removed.
    • Added the reapply_rule property, which controls what happens to the auto-remove time of a status effect in case it is re-applied.
    • Added the remove_on_hazard_leave property. This is now used for statuses like drowning, where the status effect is removed once the object leaves the hazard that was applying the status effect.
    • Added the replacement_on_timeout property, which lets a status effect be swapped for another when its time runs out.
    • Added the state_change_type and state_change_name properties, so status effects can change the object's state when applied.
    • Added the shaking_effect property, which makes the object shake left and right while under the effects of the status effect.
    • Object types, in their hazard vulnerability data and spike damage vulnerability data, can now be told to receive a status effect when interacting with one.
    • Plus other changes noted below in the compatibility warnings.
  • New mob script actions:
    • load_focused_mob_memory and save_focused_mob_memory let the currently focused object be saved into a memory, and be recalled later. This allows the object to keep track of more than one object at a time, although it still only focuses on one at a time.
    • link_with_focused_mob lets the object be linked to whatever object it has in focus.
    • hold_focused_mob lets the object hold whatever object it has in focus, much like how a leader holds a Pikmin.
    • throw_focused_mob lets the focused object be thrown in an arc towards a destination.
    • store_focus_inside lets the object store another one inside, to be released later.
    • release_stored_mobs lets the object release any other object that is stored inside.
    • get_angle lets the script obtain the angle between two sets of coordinates.
    • get_coordinates_from_angle lets the script obtain the X and Y coordinates calculated from a given angle and distance.
    • get_distance lets the script obtain the distance between two sets of coordinates.
    • get_floor_z lets the script obtain the Z coordinate of the floor at the specified coordinates.
    • set_can_block_paths lets the object block or stop blocking the paths of others.
    • set_flying lets the object move in mid air at will, unaffected by gravity.
    • set_radius lets the object's radius be changed.
    • set_shadow_visibility lets the object's shadow be hidden.
    • follow_path_to_absolute lets the object follow path stops to get to the specified coordinates.
    • follow_path_randomly lets the object follow path stops to get to a random stop in the area.
  • Other mob script changes:
    • The move_to_absolute and move_to_relative actions can now use an optional Z coordinate argument.
    • The turn_to_absolute and turn_to_relative actions can now either use an angle argument, or coordinate arguments.
    • Added group_task_power and angle to the get_info action, so a group task object can obtain the current "Pikmin power" applied to it, and so an object can obtain its current angle, respectively.
    • The on_hitbox_touch_a_n event used to trigger regardless of the first hitbox's type. This has now been fixed, and on_hitbox_touch_n_n was created for when the both hitboxes are of the normal type.
  • Group task object type improvements:
    • Added the flying_pikmin_only property.
    • Added the spots_z property to allow control of the vertical position of the contribution spots.
    • Added the carrying value to the worker_pikmin_pose property (though it doesn't change anything yet).
  • New object type properties:
    • acceleration controls the object type's acceleration.
    • has_group lets the object have a group of other objects (usually Pikmin) following it.
  • New game configuration properties:
    • no_pikmin_color controls the color the cursor shows when no Pikmin are available.
    • throw_max_dist and whistle_max_dist control the maximum distances at which the leader can throw or whistle.
  • Minor editor tweaks.
    • In the animation editor's sprite or animation picker, the buttons now shows thumbnails of the sprite or the animation's first frame.

0.18.0

February 8th, 2020 – the "missing object types" update, part 2

Changes

  • New mob categories:
    • Group tasks:
      • These are objects that require multiple Pikmin working on them at the same time, and once a certain combined power is reached, something happens, like the object moves or disappears. Useful for cardboard boxes or electrodes.
      • Each object has a series of spots, organized in rows, for Pikmin to stand on. When a Pikmin gets near, it goes to its spot and stays there doing some pose.
      • It's possible to configure the number of spots, the spacing between them, how many rows of spots exist, how the Pikmin pose when they're "working", how a Pikmin contributes, and a speed bonus for any excess contribution power (applicable when the object is moving).
      • The script for these objects is custom, and when the power goal has been met or lost, a message is sent to the object.
      • One of the contribution methods is pushing strength. As such, each Pikmin type now has a push_strength parameter.
    • Scales:
      • These measure the weight applied on top of them, and use it to do stuff. Useful for seesaw blocks or crushable paper bags.
      • Leaders, Pikmin, enemies, etc. count to the weight as long as they are on top of the object. Pikmin being held by other objects will weigh it accordingly.
      • For crushable paper bags to work, mobs can now also change their own height with the set_height script action.
      • Above the object, a number will appear signifying the weight. Optionally, a goal number can be specified, making a fraction show up instead.
    • Tracks:
      • These can have other objects ride on them, transporting the object from point A to point B. Useful for climbing sticks and slides.
      • Pikmin and/or leaders can ride them once they touch the mob's center. Then, they travel from body part 1 to 2 to 3, and so on until the end.
      • Pikmin and leaders can also optionally be whistled out of them.
    • Bouncer:
      • Any Pikmin or leader that lands atop a bouncer will be bounced away to another location. Useful for Bouncy Mushrooms and geysers.
      • The destination is whatever its linked mob is, meaning it can be easily changed in the area editor.
  • Added the following objects to the pre-packaged content (though they aren't included in any area):
    • Clog.
    • Cardboard box. (Thanks Gaming98)
    • Seesaw block.
    • Paper bag (the crushable type).
    • Climbing stick (already built), and climbing stick pile (needs to be built).
    • Bouncy Mushroom.
    • Electrical wire.
    • Spotcap and Kingcap. (Thanks Neo)
    • A dummy object to help some other objects work.
  • Object script improvements:
    • You can now use variables in script actions. Example: set_timer $extra_time.
    • As a result of the above, some complex actions got split and simplified, some redundant actions got removed, and the whole scripting system was made more robust (see the compatibility warnings).
    • New actions: goto and label. When a goto action is run, it jumps to the equivalent label action. Useful for loops.
    • New action: order_release. Use this to get a tool to order the Pikmin that is holding it to release it. (Thanks Helodity)
    • New action: set_huntable. If this is false, an object cannot be hunted down. Useful for Pellet Posy sprouts, for instance.
    • New action: drain_liquid. Drains the liquid of the sector the object is in, if it has any hazard with an associated liquid. It also affects neighboring sectors with liquid. This drain takes place over the course of two seconds.
    • New action: set_height. An object's height can now be controlled by the script, and is by default whatever the object type specifies.
    • New action: set_sector_scroll. With this, an object can change the scrolling speed of the sector it is on. A scrolling sector shifts its floor texture over time, and moves its objects in that direction too.
    • New action: get_focus_var. Use this to obtain the value of a variable belonging to the focused object.
    • New action: send_message_to_focus. This sends a message to the currently focused object, if any.
    • New action: print. It writes some text to the top of the screen, even the value of variables. Use this to help you debug your scripts.
    • New events: on_weight_added and on_weight_removed, triggered when weight is placed on top of, or removed from the object.
    • New event: on_hitbox_touch_a_n, triggered when an attack hitbox of this object touches a normal hitbox of another object. The opposite doesn't exist since it's reserved by the engine for controlling attack damage.
    • focus now has the link keyword, to make an object focus on whatever object it is linked to.
  • Changed the way combat works:
    • Objects are now of a given "target type" (identified by the object type). They are also only able to hunt down and/or hurt objects of specific target types.
    • Teams only serve to create friendships. Objects working in the same team cannot hunt down or hurt each other.

      • It has been split into move_to_absolute, move_to_relative, move_to_target.

      • It has been split into turn_to_absolute, turn_to_relative, and turn_to_target.

    • Objects in this category represent drops of some liquid on the floor, and can be drunk by other objects. Use this for nectar drops, spray nectars, etc.
    • You can specify whether Pikmin or leaders drink it, and what the effects are when they are done drinking. Maturing the Pikmin and increasing a spray counter are classic, but I've also made it so you can give the drinking object a status effect.

  • New category: Converter.
    • Objects in this category represent things that can convert any Pikmin thrown in into a seed of a Pikmin of another type. Use this for Candypop Buds.
    • You can specify what Pikmin types this object can convert to, the time between type changes, how many converted Pikmin can come out at most, how many Pikmin are ejected for every Pikmin inserted, if Pikmin of the same type as the conversion count towards the limit, how many Pikmin can be inside at once before it performs a conversion, and how long it takes before it closes and converts automatically.
  • New category: Resource.
    • Objects in this category belong to a pile, and can be carried by Pikmin and delivered somewhere in order to cause some effect. Use this for bridge shards, gold nuggets, Burgeoning Spiderwort berries, etc.
    • Among other things, you can specify where they must be delivered, what happens when they are dropped (if they vanish, and how quickly they do so), and when they are delivered (damage the receiving mob, increase the number of spray ingredients, or add points to the score).
    • Due to this, sprays can now be concocted by gathering up ingredients.
  • New category: Resource pile.
    • Objects in this category represent piles of resource objects. Use this for piles of bridge shards, piles of gold nuggets, Burgeoning Spiderworts, etc.
    • Among other things, you can specify how the piles recharge (if they do), if the number of resources should show up above it, and if it should disappear when it's empty.

Updating older content

If you have created any custom content from this list, you must make the appropriate corrections noted here if you want it to work on 0.10. If not, you can forget this section.

This section is only for content created on version 0.9; if you created content in a version before that one, please port the content version by version. Before you start, I recommend you create a backup of your content. I've made a tool that'll convert some of the files – you'll have to do the rest by hand. To use the tool, go here, and download the page as an HTML file. Then, open said HTML file with your browser of choice, and follow the instructions there.

  • Main config file:
    • cursor_spin_speed now uses degrees instead of radians.
  • All animations:
    • frames -> sprites.
    • frame_instances -> frames.
    • hitboxes -> body_parts.
    • hitbox_instances -> hitboxes.
    • file_x = <x> + file_y = <y> -> file_pos = <x> <y>.
    • file_w = <w> + file_h = <h> -> file_size = <w> <h>.
    • game_w = <w> + game_h = <h> -> game_size = <w> <h>.
    • offs_x = <x> + offs_y = <y> -> offset = <x> <y>.
  • Areas:
    • guide_x = <x> + guide_y = <y> -> reference_pos = <x> <y>.
    • guide_w = <w> + guide_h = <h> -> reference_size = <w> <h>.
    • guide_file_name -> reference_file_name.
    • guide_alpha -> reference_alpha.
    • Any enemies in the "Misc." mob category need to be changed to the "Custom" category.
    • Bridges now belong to the "Bridge" category. It's better to delete the existing bridge objects and create them again with the editor.
  • Particle generators:
    • speed -> total_speed.
    • speed_deviation -> total_speed_deviation.
    • speed_x = <x> + speed_y = <y> -> speed = <x> <y>.
    • speed_x_deviation = <x> + speed_y_deviation = <y> -> speed_deviation = <x> <y>.
    • x_deviation = <x> + y_deviation = <y> -> pos_deviation = <x> <y>.
    • angle and angle_deviation now use degrees instead of radians.
  • Mob config files:
    • rotation_speed now uses degrees instead of radians.
    • throw_height_mult -> throw_strength_mult (Pikmin and leaders only).
  • Mob scripts:
    • Mob types have lost the sight radius, near radius, etc. in favor of a detailed "reach" system:
      • Each mob type can have any number of reaches – a reach is defined by 1 or by 2 pie-slices. If an opponent is inside these pie-slices, they're considered "in reach".
      • The mob script can set what the active reach for "new" opponents is, and what the active reach for the focused opponent is.
      • For instance, a Red Bulborb, while chasing, has its "near" reach set to a pie-slice in front of it; if a Pikmin goes inside this pie-slice it will trigger an attack. Meanwhile, its "far" reach is a circle around it, such that it can always chase around the focused mob, until a certain distance.
  • Pikmin:
    • Pikmin attack intervals are no longer controlled by an attack_interval parameter, but rather, the attack connects when one frame of animation sends a "signal".
  • Sprays file:
    • angle and angle_range now use degrees instead of radians.
  • Tools config file:
    • coordinates tool -> geometry_info tool
  • HUD config:
    • There is now a positions block, that holds the coordinates of all elements, and a files block, where you specify what textures to use for the different elements.
    • 0 0 0 0 can now be used for elements you don't want.
  • Folders:
    • Game_data/Types/Misc. -> Game_data/Types/Custom.
    • Bridge type is now a thing, as is the bridge category of mobs.
