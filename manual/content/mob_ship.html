A ship type refers to any vessel that the Pikmin can deliver treasure, berries, etc. to. They may also house Pikmin inside, and leaders may also heal on them.

Examples of ship types include the S.S. Dolphin, Hocotate ship, the Hocotate ship's pod, etc.

To create a new type of ship, please follow the making object types tutorial.

Overview

Ships have a control point and a receptacle. The control point is on the floor, and is what the leader stands on to interact with the ship. It is also the spot Pikmin deliver objects to. The receptacle is the part of the ship that actually receives the object. When an object is delivered, it gets sucked in via a beam from the control point to the receptacle.

Ships can also store Pikmin inside, like Onions, though they are incapable of generating Pikmin seeds.

Ship type objects are hardcoded in the engine to sit there and accept delivered items, to heal leaders if possible, and to allow the player to manage the Pikmin inside if possibe.

Properties

Common

  • All of the common object type properties.

Recommended

can_heal If true, that means that leaders can heal their spacesuits using this ship, like the S.S. Dolphin. Boolean false
control_point_radius Radius of the control point. Number 45
control_point_offset X and Y position of the control point. This uses relative coordinates. Coordinates 0 0
control_point_offset X and Y position of the receptacle. This uses relative coordinates. Coordinates 0 0

Others

leg_body_parts Names of the body parts that make up the ship's "legs", separated by semicolon. This makes more sense if you follow the logic of the way an Onion uses this property. List
pikmin_types Internal names of the types of Pikmin this ship can manage, separated by semicolon. Leave empty if the ship cannot manage Pikmin. List
pikmin_enter_speed When a Pikmin is ordered to enter the ship, it will climb up a "leg" (see above). This indicates the speed, in ratio per second. See the track object category for more information on what this means. Number 0.7
pikmin_exit_speed Same as pikmin_enter_speed, but for popping out of a ship "leg". Number 2.0

Animations

A ship type needs to have the following animations:

  • idling: Idling; used for everything.

Sounds

A ship type needs to have the following sounds:

  • beam: Beam being activated when an object is delivered.
  • pikmin_entry: A Pikmin enters.
  • pikmin_exit: A Pikmin exits.
  • reception: Finished receiving an object.

Script

Ship types have hardcoded scripts, so the only thing you can do is enhance them.

Some script variables control special behavior, like the ship's starting Pikmin count. To know which ones exist, you can go to the area editor, select a ship, and read the tooltip of each variable.
