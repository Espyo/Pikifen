The following is an explanation of the player options in Pikifen.

All options saved by the game can be found in the user_data/options.txt file. You can edit this data file manually, though it's more comfortable to try to change these options in the options menu, or in the respective editor's options menu. Some options can't be found in either of those, since they are too niche or complex to show in-game, and need to be changed manually. Some others can be changed in-game, but you may want more customized values, in which case you'll have to edit them by hand too. If you want to find options that affect performance, just use your browser's "find" feature to search for performance in this page! The following options exist, organized into different blocks:

Advanced

Advanced options that most players will not need to change.

Block: advanced

draw_cursor_trail If false, the faint trail that follows the cursor will be removed, which could help with performance. Boolean true
engine_developer Whether you are a developer of Pikifen. This basically removes the warning that shows up when you try to edit the base pack. Boolean false
fps Framerate cap. The higher it is, the smoother the gameplay is, but the heavier on performance the engine will be. The minimum is 30 FPS. Number 60
joystick_max_deadzone Tilting the joystick this far or more will be considered the same as holding the joystick fully in that direction. If you tilt your analog stick fully, but realize that the engine is responding like if you only tilted it a bit, try lowering this value. 0.9 (90%) is a good value for most cases. Number 0.9
joystick_min_deadzone Tilting the joystick this far or less will be considered the same as keeping the joystick in neutral. If your analog stick is wobbly, and you see actions happening in-game even thought you're not moving the analog stick, try increasing this value. 0.2 (20%) is a good value for most cases. Number 0.2
max_particles Maximum number of particles to have on the world at once. If more particles than this need to be created, older particles are deleted. The more particles active at once, the worse for performance. Number 1000
mipmaps true to enable mipmaps, false to disable them. Enabling them is recommended, and increases VRAM usage for slightly less GPU usage. Turn it off if you need RAM. Might affect performance. Boolean true
p1_mouse_moves_cursor Whether player 1's leader cursor matches the mouse cursor. Boolean true
p2_mouse_moves_cursor Whether player 2's leader cursor matches the mouse cursor. Boolean false
p3_mouse_moves_cursor Whether player 3's leader cursor matches the mouse cursor. Boolean false
p4_mouse_moves_cursor Whether player 4's leader cursor matches the mouse cursor. Boolean false
smooth_scaling If true, images that need to shrink or grow big will do so smoothly. Otherwise it will look pixelated, but it might help with performance. Boolean true
window_position_hack For some users, the game window does not appear, although the engine is definitely running fine. This is because it somehow gets placed out of the operative system's screen. Set this to true to force it to appear inside the screen. Boolean false
zoom_medium_reach Same as the zoom_closest_reach game configuration property, but for the medium zoom level. Number 630

Animation editor

Options pertaining to the animation editor.

Block: animation_editor

bg_path Full path to the background texture for the animation editor, if any. Text
history List of recently-opened content. This is a semicolon-separated list of pairs. The first item in the pair is the content's path, and the second is its name. Text

Area editor

Options pertaining to the area editor.

Block: area_editor

advanced_mode If true, the area editor will use an advanced interface mode. Boolean false
backup_interval How often the area editor creates a backup of the current area, in seconds. Number 120
grid_interval Space between each grid cell in the area editor. Number 32
history List of recently-opened content. This is a semicolon-separated list of pairs. The first item in the pair is the content's path, and the second is its name. Text
selection_transformation If true, a transformation widget will appear around the selection. Boolean false
show_circular_info If true, the radius and number of vertexes of a circular sector will be shown when dragging one. Boolean true
show_edge_length If true, the length of nearby edges will be shown, when dragging them or drawing them. Boolean true
show_path_link_length If true, the length of nearby path links will be shown, when dragging them or drawing them. Boolean true
show_territory If true, the territory radius of an object will appear around it, when selected. Boolean true
snap_mode Cursor snap mode used. Number 0
snap_threshold The cursor only snaps when it is within these many pixels of something that can be snapped to. Number 80
undo_limit How many steps can be saved in the undo history, at most. 0 disables the undo feature. The higher the number, the more system memory the game will use. Number 20
view_mode View mode to use. 0 for standard, 1 for wireframe, 2 for height map, 3 for brightness map. Number 0

Audio

Options about the game's audio.

Block: audio

ambiance_sound_volume Volume for in-world ambiance sound effects, from 0 to 1. Number 1
gameplay_sound_volume Volume for in-world gameplay sound effects, from 0 to 1. Number 1
master_volume Master game volume, from 0 to 1. Number 0.8
music_volume Volume for music, from 0 to 1. Number 1
ui_sound_volume Volume for UI sound effects, from 0 to 1. Number 1

Control binds

Each player's control binds. For simplicity's sake, this page will not go into detail.

Blocks: control_binds_p1 to control_binds_p4

Controls

Miscellaneous options about player controls.

Block: controls

auto_throw_mode What mode auto-throwing is set to. 0 is off, 1 is via holding, 2 is via button toggle. Number 0
cursor_speed The speed at which the cursor moves, used when controlling it without a mouse. This is in window pixels per second. Number 500

Editors

Options that are common to all editors.

Block: editors

highlight_color Color of highlighted elements in the canvas, when using a custom editor style. Color 255 255 255
mmb_pan In editors, should the middle mouse button pan (true) or reset the camera/zoom (false)? The right mouse button will do the other function. Boolean false
mouse_drag_threshold In editors, the mouse will only be considered dragged if it's moved at least these many pixels. Number 4
open_nodes In editors, these nodes have been opened by the player. Next time they visit the editor, that node will already be opened. The nodes are named after the editor and the node itself, and are separated by semicolon. Text
primary_color In editors, if the player is using a custom style, this is the primary color to use for the widgets. Color 12 12 12
secondary_color In editors, if the player is using a custom style, this is the secondary color to use for the widgets. Color 48 119 198
show_tooltips In editors, should widget tooltips appear when the player places the mouse cursor over them? Boolean true
text_color In editors, if the player is using a custom style, this is the text color to use for the widgets. Color 255 255 255
use_custom_style In editors, should a custom color style be used for the widgets? Boolean false

Graphics

Options about the game's graphics.

Block: graphics

fullscreen Is the window fullscreen (true) or windowed (false)? Note that if the specified resolution doesn't work in fullscreen, the engine will revert the option to windowed mode. Affects performance negatively or positively, in that the more pixels are drawn on-screen the higher the cost, but also because the true_fullscreen option has an impact. Boolean false
resolution Width and height of the game window. The engine was made with a small-medium resolution in mind, and a 4:3 aspect ratio, although you can change the window size to anything you want. The higher the resolution, the worse on performance. Boolean 1024 768
true_fullscreen If true, fullscreen mode will really use a fullscreen window. If false, it will use a borderless window. True fullscreen can cause issues when you Alt+Tab to a different window, as well as when a system message box pops up (like browsing for a file in an editor). On the other hand, true fullscreen can be better for performance since depending on your operative system and graphics card, the game will have exclusive access to the display. Boolean false

GUI editor

Options pertaining to the GUI editor.

Block: gui_editor

grid_interval Space between each grid cell in the GUI editor. Number 2.5
history List of recently-opened content. This is a semicolon-separated list of pairs. The first item in the pair is the content's path, and the second is its name. Text
snap Whether the cursor snaps to the grid. Boolean true

Misc.

Miscellaneous options that do not fit anywhere else.

Block: misc

cursor_cam_weight When moving the cursor, how much does it influence the movement of the camera? 0 means none, 1 means just as much as the leader's position. 0.5 is something in-between. You can use values beyond these ranges, but be warned that things will likely be weird. Number 0
leaving_confirmation_mode How the game should ask the player if they really confirm they want to leave the area. 0 is never, 1 is after 1 minute, 2 is always. Number 2
pikmin_bump_dist Leaders must be this close to idle Pikmin before they're considered "bumped" and added to the group. Number 50
show_counter_on_cursor Whether to show a standby type counter on the leader's cursor. Boolean false
show_hud_input_icons Whether to show input icons on the HUD for various GUI items or game features. Boolean true

Packs

Options about any installed packs. These do not affect the base pack.

Block: packs

disabled List of packs that are currently disabled, separated by semicolon. Text
order The order in which each pack is loaded, separated by semicolon. Text

Options pertaining to the particle editor.

Block: particle_editor

bg_path Full path to the background texture for the particle editor, if any. Text
grid_interval Space between each grid cell in the particle editor. Number 32
history List of recently-opened content. This is a semicolon-separated list of pairs. The first item in the pair is the content's path, and the second is its name. Text
